## Краткое резюме

В чате разобрана задача стабилизации FLAT-стратегии через адаптивный шаг сетки `gridStepPerc_smart`.

Цель не в максимизации пикового PnL, а в расширении прибыльного плато вокруг базового шага, снижении просадки и уменьшении рваных месячных результатов.

Ключевой переход: оценивать не одиночный лучший PnL, а устойчивое плато из 3 и более соседних значений шага. Также предложена новая формула `v5 Magnet`, которая притягивает шаг к базовому `gridStepPerc`, например 0.27.

## Идеи

- Оценивать адаптивный шаг не по одиночному максимуму PnL, а по ширине и устойчивости плато.
- Плато считать значимым, если есть минимум 3 соседних значения шага с хорошим PnL.
- Основная цель адаптивного шага: не увеличить прибыль, а стабилизировать результат и снизить DD.
- Пересчитывать `gridStepPerc_smart`:
  - при старте новой сетки;
  - после стопа одной стороны, чтобы выходная ветка работала с актуальной волатильностью.
- Для ветки выхода особенно важно не дать шагу уехать в крайности, иначе можно ухудшить V1/V2.
- Одного VI может быть недостаточно: часть месяцев может требовать учёта ADX или отдельного запрета стартов.

## Гипотезы

### Гипотеза 1. Плато важнее пика

Одиночный лучший PnL может быть артефактом. Для выбора формулы адаптивного шага нужно искать не максимум, а устойчивую область вокруг базового шага.

Проверка:
- прогонять шаги вокруг базы, например 0.25–0.31;
- искать минимум 3 соседних положительных значения;
- сравнивать не только PnL, но и MaxDD, V1–V4, V6.

### Гипотеза 2. v5 Magnet расширит плато вокруг базового шага

Если базовый шаг 0.27 даёт хороший результат, новая формула должна не агрессивно смещать шаг, а притягивать его к 0.27.

Ожидаемый эффект:
- меньше разлёт шагов между месяцами;
- меньше рваность результатов;
- лучше устойчивость в зоне 0.26–0.28;
- возможное снижение просадки за счёт меньшего числа плохих выходных веток.

### Гипотеза 3. VI без ADX не объясняет все месяцы

Старые формулы по волатильности могут сдвигать плюсы в разные стороны, но не расширять плато. Это указывает, что одна ось волатильности не всегда объясняет рынок.

Проверка:
- отдельно смотреть месяцы с низкой VI и плохим PnL;
- сравнивать с ADX;
- определить, нужен ли трендовый фильтр или запрет старта.

## Выводы

- Использовать только лучший PnL как критерий выбора формулы — ошибка.
- Нужно искать устойчивое плато, а не один удачный шаг.
- Ранее протестированные v1–v4 полезны как база сравнения, но их нельзя автоматически считать актуальными для новой базы 0.27.
- Формула v5 должна быть отдельной формулой, а не модификацией v1–v4, чтобы не смешивать результаты прошлых тестов.
- Формула должна иметь минимум параметров: желательно только `pivot` и `coef`.
- Слишком много параметров делает оптимизацию непрактичной.
- Для FLAT важны не только PnL и DD, но и распределение исходов V1–V6.

## Ошибки и решения

### Ошибка: оценка по лучшему PnL

Проблема: одиночный лучший PnL может быть артефактом конкретного месяца или конкретного шага.

Решение:
- считать плато из 3+ соседних шагов;
- отдельно учитывать MaxDD;
- отдельно учитывать V1–V4 и V6.

### Ошибка: слишком много параметров в новой формуле

Проблема: формула с параметрами `pivot`, `coef`, `k`, `p` и другими ручками практически не тестируема.

Решение:
- оставить только `SmartPivotValue` и `SmartAbsVolCoef` / `g_snap.absVolCoef`;
- максимум добавить третий параметр только при доказанной необходимости.

### Ошибка: лишнее квантование

Проблема: если квантование уже есть ниже по коду, его не нужно добавлять в строку формулы. Иначе появляется риск двойного округления и искажения теста.

Решение:
- формула считает только `g_snap.gridStepPerc_smart`;
- квантование оставлять в существующем месте кода.

### Ошибка: случайное умножение на `coef` при квантовании

Проблема: строка вида `g_snap.gridStepPerc_smart * coef / stepQuant` меняет шаг, а не просто округляет его.

Решение:

```cpp
g_snap.gridStepPerc_smart = MathFloor(g_snap.gridStepPerc_smart / stepQuant + 0.5) * stepQuant;
```

Использовать только если квантования ниже по коду ещё нет.

## Правила

- Сначала статистика, потом новая формула.
- Не оценивать формулу по одному лучшему месяцу или одному лучшему шагу.
- Плато важнее пика.
- Чем меньше параметров в формуле, тем лучше для оптимизации.
- Формулу v5 тестировать отдельно от v1–v4.
- Старые выводы по v1–v4 держать в голове, но не переносить автоматически на новую базу.
- `gridStepPerc_smart` пересчитывать:
  - при старте сетки;
  - однократно после стопа одной стороны.
- Не пересчитывать шаг бесконечно внутри выходной ветки, чтобы не получить дрожание геометрии.
- Если квантование уже есть ниже, не добавлять его в формулу.
- В коде сохранять стиль проекта: минимум новых переменных, короткая формула, комментарии рядом с логикой.

## Архитектура

### Блок Smart-режима

В текущей архитектуре FLAT снимок режима рынка хранится в `RegimeSnap`:

- `vi` — ATR100 / ATR1000;
- `adx50` — ADX(50);
- `absVol` — ATR30 / ATR365 на D1;
- `absVolCoef` — коэффициент формулы;
- `gridStepPerc_smart` — итоговый шаг сетки;
- `forcedBufferPerc_smart` — итоговый буфер;
- `trailingBufferPerc_smart` — итоговый трейлинг.

### Логика пересчёта

Базовый сценарий:

1. При старте сетки вызывается `Regime_BeginSession()`.
2. Рассчитывается `g_snap.absVol`.
3. Рассчитывается `g_snap.gridStepPerc_smart`.
4. На основе smart-шага строится абсолютный `step`.
5. Сетка использует этот `step` до завершения или до события пересчёта.

Дополнительный сценарий:

1. При стопе одной стороны вызывается пересчёт режима.
2. Новый шаг применяется для оставшейся стороны / выходной ветки.
3. Пересчёт должен быть однократным на ветку выхода.

### Связь с исходами V1–V6

Изменение `S` влияет на:

- частоту сделок;
- шанс выхода за сетку;
- размер тейка;
- длину удержания;
- вероятность V1/V2 против V3/V4;
- риск длинных сессий и аварийных выходов.

Контекст: FLAT BUY+SELL стратегия на MT5.

## Формулы адаптивного шага

### v1. Степенная

```cpp
g_snap.gridStepPerc_smart = gridStepPerc * MathPow(g_snap.absVol, g_snap.absVolCoef);
```

Сильная реакция. Может рвать плато.

### v2. Пивот-степенная

```cpp
g_snap.gridStepPerc_smart = gridStepPerc * MathPow(g_snap.absVol / SmartPivotValue, g_snap.absVolCoef);
```

Есть pivot. Реакция понятная, но может быть недостаточно гибкой.

### v3. Лог-демпфер

```cpp
g_snap.gridStepPerc_smart = gridStepPerc * MathExp((g_snap.absVolCoef / (1.0 + MathAbs(MathLog(g_snap.absVol / SmartPivotValue)))) * MathLog(g_snap.absVol / SmartPivotValue));
```

Плавнее, чем v1/v2. Эффект смещения может быть слабым.

### v4. Tanh

```cpp
g_snap.gridStepPerc_smart = gridStepPerc * MathExp(g_snap.absVolCoef * MathTanh(MathLog(g_snap.absVol / SmartPivotValue)));
```

Сдерживает дальние от pivot значения, сохраняя локальную линейность возле pivot.

### v5. Magnet

Цель: притягивать шаг к базовому `gridStepPerc`, например 0.27.

Формула:

```cpp
g_snap.gridStepPerc_smart = gridStepPerc + g_snap.absVolCoef * (MathLog(g_snap.absVol/SmartPivotValue) / (1.0 + MathAbs(MathLog(g_snap.absVol/SmartPivotValue))));
```

Комментарии к формуле:

```cpp
// Вариант 5.0
// v5 Magnet: «магнитим» шаг к базовому gridStepPerc (S0).
// Формула: S = S0 + coef * (ln(VI/pivot) / (1+|ln(VI/pivot)|)).
// pivot — опорная волатильность (при VI≈pivot S≈S0); coef — макс. дельта к S0 (на краях → ±coef).
// Насыщение x/(1+|x|) гасит крайние отклонения → расширяет плато вокруг S0.
// Пересчёт S делать на R_START и однократно на R_STOP_EDGE.
```

Важно: в коде используется `g_snap.absVol`, хотя в комментарии удобно писать VI/pivot. Лучше в финальной версии комментария уточнить, что фактически это `AbsVol/pivot`, если формула работает именно по `g_snap.absVol`. Иначе будущий читатель, то есть очередной бедный ИИ-стажёр, начнёт искать не тот индекс.

## Что проверить дальше

- Протестировать v5 Magnet на отдельных архетипах месяцев:
  - месяц с высокой волатильностью;
  - месяц с низкой волатильностью;
  - месяц со средней волатильностью.
- Затем прогнать весь период Jan–Aug / год.
- Для каждого теста собрать:
  - PnL;
  - MaxDD;
  - распределение V1/V1.5/V2/V3/V4/V5/V6;
  - ширину плато по шагам;
  - минимум PnL внутри плато;
  - месяцы, которые остаются красными.
- Проверить две стартовые настройки:
  - `SmartPivotValue = 1.20`, `SmartAbsVolCoef = 0.02`;
  - `SmartPivotValue = 1.20`, `SmartAbsVolCoef = 0.03`.
- Если плато не расширяется, проверить, не нужен ли ADX как второй фильтр.
- Если DD остаётся около 20% или выше, пройти месяцы отдельно и найти источник просадки.
- Проверить, не ломает ли пересчёт шага после стопа одной стороны дальнейшую геометрию выходной ветки.
