## 3. Технические требования Telegram

Официальная документация:

- [Telegram Stickers: Video Stickers and Emoji](https://core.telegram.org/stickers#video-stickers)
- [Encoding Video Stickers and Emoji with WEBM and VP9](https://core.telegram.org/stickers/webm-vp9-encoding)

### 3.1. Видео-стикеры WEBM

Финальный файл должен соответствовать всем требованиям:

| Параметр | Требование |
|---|---|
| Контейнер | `.webm` |
| Видеокодек | VP9 |
| Прозрачность | alpha-канал |
| Размер кадра | одна сторона ровно `512 px`, вторая — не больше `512 px` |
| Для квадратного стикера | `512 × 512 px` |
| Длительность | не более `3 секунд` |
| Частота кадров | не более `30 FPS` |
| Звук | аудиодорожки быть не должно |
| Размер файла | не более `256 KB` |
| Цикл | желательно бесшовный loop |

Нарушение любого пункта может привести к ошибке Telegram:

```text
Bad sticker
```

### 3.2. Статические стикеры

| Параметр | Требование |
|---|---|
| Формат | PNG или WEBP |
| Размер кадра | одна сторона ровно `512 px`, вторая — не больше `512 px` |
| Для квадратного стикера | `512 × 512 px` |
| Фон | прозрачный |

## 6. Создание статических PNG-стикеров

### 6.1. Рабочий процесс

1. Загрузить утверждённый референс персонажа в ChatGPT.
2. Указать точную короткую фразу.
3. Описать эмоцию и сюжет.
4. Добавить не более 1–2 смысловых предметов.
5. Сгенерировать PNG `512 × 512 px`.
6. Проверить прозрачность.
7. Проверить текст вручную.
8. Проверить отсутствие внешнего контура.
9. При необходимости выполнить отдельную постобработку PNG.

### 6.2. Главная ошибка первого этапа

Изначально часть стикеров создавалась как классические стикеры с белой обводкой. Это оказалось ошибкой:

- белая обводка мешает визуальной интеграции;
- рядом с белой обводкой часто появляется серый кант;
- после удаления фона остаются matte, halo и fringe;
- часть изображений пришлось перегенерировать.

### 6.3. Новое обязательное правило

Каждый объект должен заканчиваться своим собственным цветом. Сразу за ним начинается прозрачность.

```text
edge layer = 0 px
matte/fringe layer = 0 px
cutout layer = 0 px
```

Не использовать формулировки, которые провоцируют генератор создать классический белый внешний контур:

- sticker-style outline;
- white outline;
- white border;
- sticker cutout;
- white rim;
- halo;
- glow around silhouette.

Белый цвет допустим только **внутри** объекта как материал или деталь:

- глаза;
- металлический блик;
- лист бумаги;
- внутренняя деталь брони;
- внутренняя деталь текста.

## 7. Базовый промпт для создания статического PNG

Скопировать промпт, заменить `[ТЕКСТ]` и `[ОПИСАНИЕ СЮЖЕТА]`.

```text
# PNG-иллюстрация RoboTradingAI / SolidarTrading — v4 clean alpha

Создай квадратную PNG-иллюстрацию 512×512 для Telegram-сообщения в стиле RoboTradingAI / SolidarTrading.

Формат результата:
- PNG, 512×512 px;
- фон полностью прозрачный: alpha=0 за пределами объектов;
- композиция изолирована в центре кадра;
- больше свободного пространства вокруг главных объектов;
- только крупные читаемые элементы;
- без мелкого фонового шума.

# Текст

Точный текст на изображении:
"[ТЕКСТ]"

Требования к тексту:
- кириллица без ошибок;
- не менять буквы;
- не добавлять слова;
- крупные жирные печатные буквы;
- оранжево-жёлтый градиент;
- читаемость за счёт размера, контраста заливки и внутренней тени;
- буквы должны быть цельными цветными формами;
- вокруг букв не создавать отдельный внешний edge-layer.

# Сюжет

[ОПИСАНИЕ СЮЖЕТА]

Оставить только смысловые крупные объекты:
- главный персонаж;
- текст;
- 1–2 ключевых предмета, если они нужны по смыслу;
- 1–3 маленьких orange/cyan акцента максимум.

Убрать визуальный мусор:
- фоновые вспышки;
- случайные штрихи;
- дым;
- конфетти;
- лишние знаки;
- мелкие декоративные элементы;
- любые подложки за всей композицией.

# Главный персонаж

Мультяшный робот-помощник SolidarTrading / RoboTradingAI:
- молодой мужчина;
- короткие коричневые волосы;
- чёрные очки;
- голубые глаза;
- дружелюбное или нужное по сюжету выражение лица;
- серебристая роботизированная броня;
- cyan/голубые световые линии внутри брони;
- оранжевые круглые акценты на броне;
- clean comic cartoon illustration;
- ярко, продающе, но не мрачно;
- не реализм;
- не пластиковое 3D;
- не агрессивный супергерой.

Если есть референс, использовать из него только:
- образ персонажа;
- лицо;
- причёску;
- очки;
- броню;
- позу;
- основные крупные предметы.

Не копировать из референса:
- фон;
- подложки;
- декоративный шум;
- краевые артефакты;
- матовый слой вокруг объектов;
- светлый слой между объектом и прозрачностью.

# Alpha-edge rule

Главное техническое правило: каждый объект заканчивается своим собственным цветом, а сразу за ним начинается прозрачность.

Внешний край объекта:
- natural alpha edge;
- edge layer = 0 px;
- matte/fringe layer = 0 px;
- cutout layer = 0 px;
- объект не отделяется от прозрачности дополнительной линией;
- сглаживание только за счёт полупрозрачных пикселей того же материала, что и сам объект.

Светлые пиксели допустимы только внутри объектов, если это часть материала:
- глаза;
- металлический блик;
- бумага;
- внутренняя деталь брони;
- внутренняя деталь текста.

На внешнем alpha-edge не должно быть светлой прослойки, если она не является реальной частью самого объекта.

# Цвет и стиль

Общий стиль:
- high-energy comic cartoon;
- чистые формы;
- крупные массы;
- насыщенные orange/cyan accents;
- детальность только внутри персонажа и ключевых предметов;
- без перегруженного фона.

Текст:
- orange/yellow gradient fill;
- тёмная внутренняя тень;
- локальный внутренний контраст;
- без отдельной внешней линии по краю букв.

Персонаж:
- внутренние comic-линии допустимы;
- линии деталей брони допустимы;
- тени внутри форм допустимы;
- внешний край силуэта остаётся чистым natural alpha edge.

# Композиция

- главный объект занимает центр;
- крупные формы важнее мелких деталей;
- между объектами и краями холста оставить воздух;
- текст должен читаться в маленьком размере;
- предметы не должны перекрывать лицо и главный текст;
- изображение не должно выглядеть как рекламный баннер с фоном.

# Финальная самопроверка

Перед выдачей проверь:
1. Размер 512×512 px.
2. Фон полностью прозрачный.
3. Нет квадратной подложки.
4. Нет фоновых пятен вокруг всей композиции.
5. Главные объекты крупные и читаемые.
6. Лишние элементы удалены.
7. Текст написан точно.
8. Русские буквы не искажены.
9. Внешние края объектов — natural alpha edge.
10. Между объектами и прозрачностью нет отдельного edge-layer.
11. Светлые пиксели находятся только внутри объектов, а не на внешнем alpha-edge.
12. Иллюстрация выглядит как чистая PNG-картинка для Telegram-сообщения.
```

## 9. Подготовка зелёной технической копии для Kling AI

Перед анимацией создать отдельную техническую версию статического стикера.

### Зачем нужен зелёный фон

Kling AI должен анимировать персонажа, но не создавать сложный фон. Ровный зелёный фон облегчает последующее удаление подложки и снижает количество артефактов.

Pixelcut умеет удалять фон и без green screen. Но для нашего процесса однотонный chroma-key фон остаётся предпочтительным:

- края проще контролировать;
- меньше случайных деталей;
- проще заметить ошибки;
- меньше риск получить сложный фон после генерации;
- проще повторно обработать видео в других редакторах.

### Цвет

```text
#00FF00
```

### Промпт для создания зелёной копии

```text
Отредактируй загруженное изображение без изменения персонажа, текста и композиции.

Замени все прозрачные и пустые области на идеально ровный сплошной chroma-key green background: #00FF00.

Требования:
- зелёный фон должен полностью заполнять весь кадр;
- фон строго однотонный;
- без градиента;
- без текстуры;
- без теней;
- без бликов;
- без шума;
- без новых объектов;
- без изменения персонажа;
- без изменения текста;
- без изменения позы;
- без изменения размеров объектов;
- без белой обводки;
- без прозрачных областей.

Результат:
PNG 512×512 px с ровным зелёным фоном #00FF00.
```

### 10.2. Почему Multi-shot нужно выключать

Для стикера нужен один стабильный сюжет в одном кадре.

Multi-shot может:

- менять ракурс;
- менять фон;
- ломать текст;
- добавлять лишние сцены;
- ухудшать бесшовность цикла;
- деформировать персонажа.

### 10.3. Общий промпт для анимации

Вставить конкретное движение в блок `[ANIMATION ACTION]`.

```text
Animate the uploaded Telegram sticker as a clean 3-second seamless loop.

IMPORTANT BACKGROUND RULE:
Keep the entire background perfectly uniform solid chroma-key green: #00FF00.
The green background must fill the whole frame behind the character, text, and all objects.
Keep the background completely flat, static, and unchanged during the entire animation.
Do not add gradients, texture, shadows, glow, particles, smoke, scenery, extra objects, or lighting changes to the green background.

CHARACTER AND DESIGN PRESERVATION:
Preserve the exact character identity, face, hairstyle, glasses, armor design, colors, proportions, text, and composition from the uploaded image.
Do not redraw the character.
Do not distort the face.
Do not change the text.
Do not add new words.
Do not add new objects.
Do not crop important objects.
Do not add a white outline, border, halo, or sticker cutout around any object.

ANIMATION ACTION:
[ANIMATION ACTION]

MOTION STYLE:
Use smooth, readable, moderately exaggerated cartoon motion.
Keep the animation simple and controlled.
The first and last frames should match naturally to create a seamless loop.
The text should remain readable and stable.
No camera movement.
No zoom.
No scene transition.
No multi-shot sequence.
No audio.
```

### 10.4. Как формулировать движение

Хорошая анимация для стикера:

- простая;
- короткая;
- цикличная;
- хорошо читается в маленьком размере;
- не ломает лицо;
- не ломает буквы;
- не требует смены кадра.

Примеры:

```text
The character gives a short confident thumbs-up gesture, slightly nods once, then smoothly returns to the original pose.
```

```text
The character raises one hand in a brief greeting wave, smiles, and smoothly returns to the starting position.
```

```text
The character points at the important message with one clear gesture while the main object makes a subtle bounce, then everything returns to the original pose.
```

```text
The character briefly celebrates with a restrained fist pump while the coins make a small controlled bounce, then smoothly return to the original positions.
```

- загрузку MP4, MOV, WEBM, MKV и GIF;

- экспорт MP4 только с непрозрачным фоном;
- бесплатное тестовое использование;
- бесплатные HD-выгрузки до 720p;
- ограничение бесплатного тарифа по количеству загрузок.

### 11.2. Практический процесс

1. Скачать результат Kling AI.
2. Загрузить видео в Pixelcut.
3. Выбрать прозрачный фон.
4. Проверить края персонажа и текста.
5. Скачать результат в формате WEBM.
6. Передать WEBM в ChatGPT для финальной обработки.

### 11.3. Важное уточнение про логотип

Pixelcut заявляет экспорт без watermark.

Если в финальном WEBM остаётся нижний логотип, он обычно уже присутствовал в исходном видео Kling AI. Его нужно убрать отдельной crop-обрезкой на финальном этапе.

## 12. Финальная обработка WEBM через ChatGPT

После Pixelcut загрузить прозрачный WEBM в ChatGPT.

### 12.1. Что нужно сделать

- убрать нижний логотип crop-обрезкой;
- не вырезать значимые объекты;
- восстановить квадратный кадр при необходимости;
- сохранить прозрачность;
- убрать аудиодорожку;
- привести размер к `512 × 512 px`;
- сохранить кодек VP9;
- ограничить FPS значением не выше 30;
- сохранить длительность не более 3 секунд;
- сжать файл до размера не более 256 KB;
- проверить итоговый WEBM.

### 12.2. Готовый запрос

```text
Обработай загруженный WEBM как финальный Telegram-видео-стикер.

Задача:
1. Убери нижний логотип аккуратной crop-обрезкой.
2. Не обрезай персонажа, текст и смысловые элементы.
3. Сохрани прозрачный фон и alpha-канал.
4. Не добавляй фон.
5. Не заменяй прозрачность чёрным цветом.
6. Удали аудиодорожку полностью.
7. Приведи итоговый размер кадра к 512×512 px.
8. Используй WEBM с кодеком VP9.
9. Частота кадров — не более 30 FPS.
10. Длительность — не более 3 секунд.
11. Итоговый размер файла — не более 256 KB.
12. Верни готовый WEBM для скачивания.

После обработки отдельно проверь:
- прозрачность сохранена;
- нижний логотип удалён;
- кодек VP9;
- отсутствует аудиопоток;
- размер не превышает 256 KB;
- Telegram должен принять файл без ошибки Bad sticker.
```

### 12.3. Если логотип перекрывает важные элементы

Обычный crop может повредить композицию. Тогда есть два варианта:

1. Вернуться к зелёному PNG и увеличить свободное пространство снизу перед анимацией.
2. Использовать ручной монтаж в CapCut.

Не применять crop автоматически, если нижняя часть персонажа или текста попадает в область обрезки.

## 13. Запасной маршрут через CapCut

Сервис:

- [https://www.capcut.com/](https://www.capcut.com/)

### Возможности

CapCut можно использовать для:

- ручного удаления зелёного фона через Chroma Key;
- ручного crop;
- удаления нижнего логотипа;
- обрезки первых кадров;
- укорочения ролика;
- визуального контроля.

### Ограничение

CapCut не является оптимальным финальным маршрутом для Telegram-видео-стикера:

- прозрачность может сохраняться не в готовом WEBM VP9;
- может понадобиться промежуточный MOV с alpha;
- после конвертации alpha иногда теряется;
- фон может стать чёрным;
- требуется дополнительная обработка.

Название прозрачного режима экспорта зависит от версии и платформы CapCut. Искать вариант экспорта с alpha-каналом. Не считать обычный MP4 прозрачным форматом.

### Когда использовать CapCut

CapCut нужен только если:

- логотип нельзя убрать простым crop;
- нужно вручную удалить первые неудачные кадры;
- требуется сложная ручная коррекция;
- Pixelcut дал артефакты.

## 15. Типовые проблемы и решения

| Проблема | Вероятная причина | Решение |
|---|---|---|
| Telegram пишет `Bad sticker` | Нарушен кодек, размер, длительность, FPS, аудио или вес файла | Привести WEBM к VP9, удалить звук, ограничить 3 секундами, 30 FPS и 256 KB |
| После конвертации фон стал чёрным | Потерян alpha-канал или выбран MP4 | Вернуться к прозрачному WEBM из Pixelcut; не использовать MP4 как прозрачный формат |
| Вокруг объектов белая обводка | Генератор создал classic sticker outline, matte или fringe | Использовать clean-alpha промпт; объект должен сразу переходить в прозрачность |
| Рядом с белым контуром появился серый кант | Сохранён matte-layer или некачественная вырезка | Удалить внешний edge-layer отдельной постобработкой PNG |
| На видео остался нижний логотип | Watermark был в исходном Kling-видео | Удалить нижнюю область crop-обрезкой на финальном этапе |
| После удаления фона много артефактов | Сложный фон, движение или плохие края | Использовать ровный фон `#00FF00`; уменьшить сложность движения |
| Первые кадры выглядят плохо | Kling создал стартовый переход | Удалить первые кадры; затем повторно проверить длительность и цикл |
| Текст деформируется при анимации | Модель пытается перерисовать буквы | Просить оставить текст статичным и стабильным; упростить движение |
| Персонаж меняет лицо | Слишком сложная сцена или движение | Использовать одно простое действие, без камеры и Multi-shot |
| Видео открывается в браузере на чёрном фоне | Браузерный плеер не всегда показывает прозрачность корректно | Проверить через Telegram либо техническим анализом файла |
| После crop персонаж стал слишком крупным | Слишком большая нижняя обрезка с последующим растяжением | На этапе зелёного PNG добавить больше свободного пространства снизу |

## 16. Финальная проверка WEBM

Перед загрузкой в Telegram проверить каждый файл.

### 16.1. Чек-лист

- [ ] Файл имеет расширение `.webm`.
- [ ] Используется кодек VP9.
- [ ] Прозрачный фон сохранён.
- [ ] Нет зелёного фона.
- [ ] Нет чёрной подложки.
- [ ] Нет нижнего логотипа.
- [ ] Кадр `512 × 512 px` либо одна сторона ровно `512 px`.
- [ ] Длительность не больше `3 секунд`.
- [ ] Частота кадров не больше `30 FPS`.
- [ ] Нет аудиопотока.
- [ ] Вес файла не превышает `256 KB`.
- [ ] Первый и последний кадр не создают резкого скачка.
- [ ] Текст читается.
- [ ] Лицо персонажа не искажено.
- [ ] Telegram принимает файл без ошибки `Bad sticker`.

### 16.2. Проверка через FFprobe

При необходимости проверить свойства файла локально:

```bash
ffprobe -v error \
  -show_entries stream=codec_name,width,height,r_frame_rate:format=duration,size \
  -of default=noprint_wrappers=1 \
  sticker.webm
```

Для поиска аудиопотока:

```bash
ffprobe -v error -select_streams a -show_entries stream=codec_name -of csv=p=0 sticker.webm
```

Команда не должна вернуть аудиокодек.

## 17. Загрузка в Telegram

Использовать:

- [@Stickers](https://t.me/Stickers)

Процесс:

1. Создать новый sticker pack.
2. Загрузить финальный PNG или WEBM.
3. Назначить подходящие эмодзи.
4. Повторить для всех стикеров.
5. Опубликовать pack.
6. Проверить стикеры в Telegram на светлой и тёмной теме.

### Правило подбора эмодзи

Для каждого стикера назначать от 1 до 5 эмодзи:

- по смыслу фразы;
- по эмоции;
- с учётом юмора;
- без бессмысленного повторения денег и роста;
- без одинаковых наборов для большинства стикеров.

## 18. Организация файлов

Для каждого стикера хранить все стадии отдельно.

Пример:

```text
01_privet/
├── 01_source_reference.png
├── 02_static_clean_alpha.png
├── 03_green_background.png
├── 04_kling_video_3_source.mp4
├── 05_pixelcut_alpha.webm
└── 06_telegram_final.webm
```

Это позволяет:

- быстро исправить ошибку;
- не повторять генерацию Kling;
- менять только последний этап;
- сохранять исходное качество;
- повторно использовать статический PNG.
