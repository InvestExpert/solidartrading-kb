## Ошибки и решения

- **Ошибка / блокер:** в текущем `ATLAS crypto 6 3.py` режим `live` заявлен как рабочий, но `CreateExchange()` не передаёт `apiKey/secret`.
  - Симптом: после установки Python и пакетов бот всё ещё не готов к реальной live-торговле
  - Причина: отсутствует авторизация биржи в exchange-блоке
  - Решение: отдельная минимальная правка `CreateExchange()` и безопасное хранение ключей
  - Где проявляется: TiTan Atlas (BA) / Python / live / Binance spot

- **Ошибка архитектуры эксплуатации:** запуск бота в одном общем окружении вместе с другим ботом на VPS.
  - Симптом: возможные конфликты библиотек и неочевидные сбои
  - Причина: общий Python / общие пакеты / отсутствие изоляции
  - Решение: отдельная папка проекта и отдельный `.venv`
  - Где проявляется: Windows VPS / эксплуатация нескольких ботов

- **Ошибка эксплуатации:** надеяться только на окно RDP-терминала как на боевой режим запуска.
  - Симптом: потеря удобного live-вывода, риск остановки/потери контроля над процессом
  - Причина: ручной запуск без отдельной схемы удержания процесса
  - Решение: позже перейти на отдельный автозапуск/Task Scheduler/service-like схему
  - Где проявляется: Windows VPS / long-run live
