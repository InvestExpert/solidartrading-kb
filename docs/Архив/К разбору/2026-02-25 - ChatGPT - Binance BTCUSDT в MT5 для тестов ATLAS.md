## Краткое резюме

В чате разобран практический путь загрузки исторических данных Binance BTCUSDT в MT5 для тестирования бота ATLAS на криптовалютной истории.

Главная цель: использовать MT5 как тестовую среду для бота, который в будущем должен работать на Spot/Python через биржу. Поэтому выбран источник данных Binance Spot, а не Futures.

Были решены ключевые проблемы:
- выбор правильного источника данных Binance Vision;
- конвертация M1-свечей Binance в формат MT5;
- обход проблемы Google Sheets с микросекундными timestamp;
- создание кастомного символа `BTCUSDT_CUST`;
- импорт тиков и баров;
- исправление ошибки `тиковый объем должен быть больше нуля`;
- исправление пустого графика;
- исправление `Invalid volume`;
- поиск причины нулевых проходов оптимизации.

## Точные факты

### Источник данных

Для будущего Spot-бота на Python данные нужно брать из Binance Spot, а не Binance Futures.

Правильный путь Binance Vision для M1-свечей BTCUSDT:

```text
data/spot/monthly/klines/BTCUSDT/1m/
```

Для реальных тиков / сделок возможен путь:

```text
data/spot/monthly/trades/BTCUSDT/
```

Но для первичных тестов выбран M1, потому что:
- меньше объём данных;
- проще конвертация;
- достаточно для первичной проверки сеточного алгоритма;
- тики Binance trades требуют дополнительной обработки.

### Что выбирать на Binance Vision

Последовательность:
1. `data/`
2. `spot/`
3. `monthly/`
4. `klines/`
5. `BTCUSDT/`
6. `1m/`
7. скачать `.zip`, не скачивать `.CHECKSUM`.

Файлы `.CHECKSUM` не нужны для импорта в MT5.

### Выбор Spot vs Futures

Решение: брать Spot BTCUSDT.

Причины:
- конечная цель — Python-бот для Spot Binance;
- Spot не содержит funding;
- Spot не содержит маржинальную ликвидационную логику;
- Futures может иметь премию/дисконт к Spot;
- для сетки даже небольшие отличия цены влияют на глубину, шаги, просадку и частоту уровней.

Futures-данные нужны только если бот будет реально торговать perpetual/futures.

### Binance CSV без заголовков

Пример строки Binance kline:

```text
1735689600000000,93576.00000000,93610.93000000,93537.50000000,93610.93000000,8.21827000,1735689659999999,768978.75522470,2631,3.95157000,369757.32652890,0
```

Структура колонок:

1. `open_time`
2. `open`
3. `high`
4. `low`
5. `close`
6. `volume`
7. `close_time`
8. `quote_asset_volume`
9. `number_of_trades`
10. `taker_buy_base_asset_volume`
11. `taker_buy_quote_asset_volume`
12. `ignore`

Для MT5 Bars нужны:
- `Date`
- `Time`
- `Open`
- `High`
- `Low`
- `Close`
- `TickVolume`
- `Volume`

### Timestamp

В данных Binance встретился timestamp:

```text
1735689600000000
```

Это 16 цифр, значит микросекунды.

Правило:
- 13 цифр = миллисекунды;
- 16 цифр = микросекунды.

Google Sheets использовать нельзя для 16-значных timestamp, потому что он округляет большие числа и теряет точность. На скрине timestamp превратился в одинаковые значения вида `1735690000000000`, из-за чего время становится некорректным.

Вывод: конвертацию Binance timestamp лучше делать через PowerShell / `.bat`, а не через Google Sheets.

### Формат итогового MT5 Bars CSV

Рабочий формат:

```text
Date,Time,Open,High,Low,Close,TickVolume,Volume
2025.01.01,00:00:00,93576.00000000,93610.93000000,93537.50000000,93610.93000000,1,8.21827000
2025.01.01,00:01:00,93610.93000000,93652.00000000,93606.20000000,93652.00000000,1,12.14029000
```

MT5 потребовал, чтобы `TickVolume > 0`.

Если `TickVolume = 0`, появляется ошибка:

```text
тиковый объем должен быть больше нуля
```

### TickVolume

Решение для баров: поставить `TickVolume = 1`.

Вывод:
- если стратегия не использует объёмные индикаторы и не торгует по объёму, `TickVolume = 1` не должен влиять на результат;
- если стратегия использует `Volume`, `tick_volume`, `iVolume`, OBV, MFI, VWAP или фильтры по объёму, нужно использовать реальные значения, например `number_of_trades`.

### Тики для MT5

Для графика MT5 рабочей оказалась схема тиков **2 тика на минуту**:
- тик на `HH:MM:00.000` по Open;
- тик на `HH:MM:59.999` по Close.

Итоговый формат tick CSV:

```text
Date,Time,Bid,Ask,Last,Volume,Flags
```

Для простого моделирования:
- `Bid = Ask = Last = Open` для первого тика минуты;
- `Bid = Ask = Last = Close` для второго тика минуты;
- `Volume = 1`;
- `Flags = 30`.

### Custom symbol

Изначально использовалось имя `BTCUSDT`, но возник риск конфликта с серверным/брокерским символом.

Решение: создать уникальный custom symbol:

```text
BTCUSDT_CUST
```

Путь:

```text
Custom\Crypto\BTCUSDT_CUST
```

### Основные параметры custom symbol

Рабочая логика для тестов:

```text
Symbol = BTCUSDT_CUST
Path = Custom\Crypto\BTCUSDT_CUST
Description = Bitcoin vs Tether (Spot) [CUSTOM]
Category = Crypto
Exchange = Binance
Digits = 2
Point = 0.01
TickSize = 0.01
TickValue = 0.01
ContractSize = 1
CurrencyBase = BTC
CurrencyProfit = USD
CurrencyMargin = USD
Spread = 0
StopsLevel = 0
FreezeLevel = 0
VolumeMin = 0.01
VolumeStep = 0.01
VolumeMax = 1000
VolumeLimit = 0
Sessions = 24/7
Chart mode = Bid
```

USD выбран вместо USDT, потому что терминалу MT5 не нужен отдельный курс USDT для тестов. Для тестовой модели достаточно считать прибыль и маржу в USD.

### Объёмы MT5 в JSON

В MT5 экспорт объёма не обязательно выглядит как `0.01`. В экспортированном JSON MT5 использует пары `Volume*` и `Volume*Ext`.

Для `0.01 / 0.01 / 1000` были зафиксированы значения:

```json
"VolumeMin" : "100",
"VolumeMinExt" : "1000000",
"VolumeStep" : "100",
"VolumeStepExt" : "1000000",
"VolumeMax" : "10000000",
"VolumeMaxExt" : "100000000000",
"VolumeLimit" : "0",
"VolumeLimitExt" : "0"
```

Вывод: если JSON вручную писать как `"VolumeMin": "0.01"`, MT5 может не применить это поле. Надёжнее:
1. вручную выставить объёмы в UI;
2. сделать экспорт;
3. использовать экспорт как эталон структуры.

### VolumeLimit

Решение: `VolumeLimit = 0`.

Смысл:
- `VolumeLimit = 0` означает отсутствие лимита суммарного объёма по направлению;
- для сеточного бота это безопаснее;
- если поставить положительный лимит, сетка может перестать добавлять уровни и давать ошибки ограничения объёма.

### Сопоставление Forex 0.01 и BTC-лота

Нужно приравнивать не “лот”, а номинал позиции.

Для GBPAUD:
- 1 лот = 100 000 GBP;
- 0.01 лота = 1 000 GBP;
- при GBPAUD ≈ 1.90 это ≈ 1 900 AUD;
- при AUD ≈ 0.65 USD это ≈ 1 235 USD.

Для BTC:
- BTC ≈ 90 000 USD;
- чтобы получить позицию ≈ 1 200 USD:

```text
BTC lot = 1200 / 90000 ≈ 0.0133 BTC
```

Вывод:
- 0.01 BTC ≈ 900 USD позиции при BTC 90 000;
- 0.01–0.015 BTC сопоставимо с Forex 0.01 GBPAUD по размеру позиции;
- для тестов принято использовать `0.01`, если `1 lot = 1 BTC`.

### Ошибка -> причина -> решение

| Ошибка / симптом | Причина | Решение |
|---|---|---|
| Google Sheets показывает `1,73569E+15` | timestamp слишком большой | не использовать Google Sheets для timestamp |
| timestamp стал одинаковым в соседних строках | потеря точности Google Sheets | делать конвертацию через `.bat` / PowerShell |
| `тиковый объем должен быть больше нуля` | TickVolume = 0 или отсутствует | добавить `TickVolume = 1` |
| Бары импортируются красным | MT5 не принимает строки / невалидный tick volume | добавить TickVolume > 0 |
| График пустой при наличии истории | не тот символ / нет тиков / не построился график-кэш | использовать `BTCUSDT_CUST`, импортировать тики 2 на минуту |

| `Unsupported filling mode` | режим заполнения ордера не поддерживается символом | включить поддерживаемые filling modes или править `type_filling` в коде |
| Оптимизация по нулям | первая сделка не открыта | смотреть журнал, искать reject первого BUY |
| 0 трейдов, 0 прибыль, 0 просадка | не было ни одной сделки | не считать это результатом стратегии |

## Идеи

### Использовать MT5 как промежуточный тестер для будущего Python Spot-бота

Суть: загрузить Binance Spot-историю в MT5 через custom symbol и использовать существующий MT5-бот/оптимизатор для первичного подбора параметров перед переносом/валидацией в Python.

Польза:
- быстро использовать текущий MT5-оптимизатор;
- не ждать полной готовности Python-бэктестера;
- получить первичные диапазоны параметров на BTCUSDT.

Ограничение:
- MT5-модель тиков искусственная;
- комиссии Binance и spot-исполнение нужно учитывать отдельно;
- итоговую проверку всё равно делать в Python.

### Генерировать синтетические тики из M1

Суть: из каждой M1 свечи делать два тика:
- Open в начале минуты;
- Close в конце минуты.

Это позволяет MT5 строить график и запускать тестер, даже если нет реальных тиков.

Ограничение:
- внутриминутные high/low не воспроизводятся;
- для трейлинга внутри минуты это может искажать результат;
- подходит для первичной оценки, не для финальной точности.

### Использовать `number_of_trades` как TickVolume

В текущем тесте TickVolume поставлен 1. Альтернатива: брать из Binance CSV поле `number_of_trades` и использовать его как TickVolume.

Статус: идея для улучшения качества данных, если объёмные индикаторы или фильтры будут использоваться.

## Гипотезы

### Гипотеза 1: Spot-данные лучше Futures-данных для будущего Spot-бота

Суть: если бот будет торговать Spot Binance, то тестировать его нужно на Spot BTCUSDT, а не на Futures BTCUSDT.

К чему относится: ATLAS crypto / Python Spot.

Чем проверить:
- сравнить Spot и Futures BTCUSDT на кризисных участках;
- проверить расхождения по глубине сетки, DD, количеству входов/выходов.

Статус: логически подтверждена, но численно не проверялась в этом чате.

### Гипотеза 2: TickVolume = 1 не влияет на результат сеточника

Суть: если стратегия не использует объёмы, то фиктивный TickVolume нужен только для MT5 и не влияет на сделки.

К чему относится: MT5 custom data / ATLAS.

Чем проверить:
- прогнать один и тот же тест с TickVolume = 1 и TickVolume = number_of_trades;
- сравнить количество сделок, PnL, DD.

Статус: частично подтверждена логикой, не проверена тестом.

### Гипотеза 3: Нулевые проходы оптимизации вызваны reject первой сделки

Суть: если в оптимизации результат 0, прибыль 0, трейдов 0, то это не результат стратегии, а технический сбой старта: первая BUY не открылась.

К чему относится: MT5 Strategy Tester / ATLAS BA.

Чем проверить:
- взять параметры нулевого прохода;
- запустить одиночный тест;
- включить журнал;
- искать `Invalid volume`, `Unsupported filling mode`, `no money`, `market closed`, `trade disabled`.

Статус: подтверждена логом для одного случая (`failed market buy 0 ... [Invalid volume]`).

## Выводы

1. Для будущего Spot-бота нужно использовать Spot-историю Binance, а не Futures.
2. Binance Vision удобен для загрузки M1-свечей без API и лимитов.
3. Google Sheets нельзя использовать для микросекундных timestamp Binance, потому что он теряет точность.
4. Для MT5 Bars нужен положительный TickVolume.
5. При импорте custom data лучше использовать уникальное имя символа `BTCUSDT_CUST`, а не `BTCUSDT`.
6. Для пустого графика при наличии истории нужно проверить не только бары, но и тики.
7. Рабочая схема для графика: импортировать тики 2 на минуту.
8. `VolumeLimit` для сеточного бота должен быть 0.
9. Нулевой проход оптимизации нельзя считать стратегическим результатом.
10. 0 трейдов = диагностика технической причины, чаще всего отказ первого ордера.
11. 0.01 BTC при контракте 1 BTC/lot является близким по номиналу к Forex 0.01 на GBPAUD.
12. Финальная проверка должна идти в Python/Spot, потому что MT5-среда всё равно остаётся приближением.

## Ошибки и решения

### Ошибка: выбран `BTCUSD` вместо `BTCUSDT`

Симптом: пользователь оказался в папке `BTCUSD`.

Причина: похожие тикеры в Binance Vision.

Решение: перейти именно в:

```text
data/spot/monthly/klines/BTCUSDT/1m/
```

Где проявляется: выбор источника данных.

### Ошибка: Google Sheets округлил timestamp

Симптом:
- timestamp отображался как `1,73569E+15`;
- после форматирования соседние строки стали одинаковыми.

Причина:
- Google Sheets хранит только ограниченное число значащих цифр;
- 16-значные микросекундные timestamp Binance искажаются.

Решение:
- не использовать Google Sheets для конвертации времени;
- использовать PowerShell / `.bat`.

### Ошибка: MT5 требует тиковой объём больше нуля

Симптом:

```text
тиковый объем должен быть больше нуля
```

Причина:
- в импортируемом Bars-файле TickVolume отсутствует или равен 0.

Решение:
- добавить колонку `TickVolume`;
- поставить `TickVolume = 1`;
- импортировать с включённой галкой `Тиковые объемы`.

### Ошибка: график пустой, хотя бары/тики есть

Симптом:
- custom symbol открыт;
- в вкладке тиков данные есть;
- график чёрный.

Промежуточные проверенные причины:
- не только имя символа;
- не только красная подсветка;
- не только сервер брокера;
- не только таймфрейм.

Рабочее решение:
- использовать уникальный символ `BTCUSDT_CUST`;
- импортировать тики;
- лучше 2 тика на минуту;
- проверить, что график строится на M1.

### Ошибка: `Invalid volume`

Симптом из журнала:

```text
failed market buy 0 BTCUSDT_CUST [Invalid volume]
```

Причина:
- бот отправил `buy 0`;
- или спецификация символа имеет неверные Min/Step/Max объёмы;
- или `GetProgLot()` / расчёт лота дал 0.

Решение:
- выставить Min volume = `0.01`;
- Step = `0.01`;
- Max = `1000`;
- проверить, что в журнале `lot` не равен 0;
- для изоляции временно отключить ProgLot и поставить фиксированный лот `0.01`.

### Ошибка: `Unsupported filling mode`

Симптом:

```text
failed market buy ... [Unsupported filling mode]
```

Причина:
- режим заполнения ордера, который отправляет EA, не поддерживается спецификацией custom symbol.

Решение:
- включить поддерживаемые filling modes в спецификации;
- если нельзя через UI, править код отправки ордера и задавать подходящий `request.type_filling`.

### Ошибка: нулевые проходы оптимизации

Симптом:
- `Результат = 0`;
- `Прибыль = 0`;
- `Всего трейдов = 0`;
- `Просадка = 0`.

Причина:
- сделок не было;
- чаще всего первая сделка не открылась из-за reject.

Решение:
- брать параметры нулевого прохода;
- запускать одиночный тест;
- смотреть журнал;
- не включать такие проходы в оценку стратегии как валидные.

## Правила

1. Если будущий бот будет Spot, исторические данные для MT5-теста брать из Spot, а не Futures.
2. Для Binance M1 использовать `klines`, а не `trades`, если нужна простая первичная проверка.
3. `.CHECKSUM` файлы не нужны для импорта.
4. Не использовать Google Sheets для 16-значных timestamp Binance.
5. Для MT5 Bars обязательно давать `TickVolume > 0`.
6. Для custom symbol использовать уникальное имя, например `BTCUSDT_CUST`.
7. Для сеточного бота `VolumeLimit` оставлять 0.
8. Нулевые проходы оптимизации с 0 трейдов исключать из анализа стратегии.
9. Проверку reject-ошибок делать через одиночный тест и журнал.
10. Сопоставлять Forex и BTC по номиналу позиции, а не по числу лотов.
11. Если стратегия не использует объёмы, фиктивный TickVolume допустим только как технический костыль для MT5.
12. Если стратегия использует объёмы, TickVolume нужно строить из реальных данных, например `number_of_trades`.


### Сопоставление BTC-лота с Forex-номиналом

```text
BTC lot = Forex nominal USD / BTC price
```

Пример:

```text
Forex nominal ≈ 1200 USD
BTC price ≈ 90000 USD
BTC lot = 1200 / 90000 ≈ 0.0133
```

Практическое решение:

```text
BTC lot = 0.01
```

### Стоимость BTC-позиции

```text
Position nominal = BTC lot * BTC price
```

Пример:

```text
0.01 * 90000 = 900 USD
```

## Риски, ограничения и дисклеймеры

1. M1-данные не дают реальную внутриминутную траекторию.
2. Синтетические тики 2 на минуту не воспроизводят high/low внутри свечи.
3. Трейлинг и сетка могут отличаться от реального тикового поведения.
4. TickVolume = 1 допустим только если стратегия не использует объёмы.
5. MT5 custom symbol не является точной моделью Binance Spot.
6. Комиссии Binance нужно отдельно моделировать, если они важны.
7. Spread = 0 даёт идеализированный тест.
8. Нулевые проходы оптимизации нужно исключать или диагностировать отдельно.
9. Нельзя переносить выводы MT5 на реальный Python Spot без финальной проверки в Python.
10. Futures-данные могут исказить результат для Spot-бота.

## Приложение: convert_mt5.bat

Назначение: из `BTCUSDT_1y_raw.csv` создать MT5 Bars CSV с `TickVolume = 1`.

```bat
@echo off
setlocal EnableExtensions

set "IN=BTCUSDT_1y_raw.csv"
set "OUT=BTCUSDT_1y_mt5_tick.csv"

if not exist "%IN%" (
  echo ERROR: Input file not found: %IN%
  exit /b 1
)

powershell -NoProfile -ExecutionPolicy Bypass -Command ^
  "$in='%IN%'; $out='%OUT%';" ^
  "$sr = [System.IO.StreamReader]::new($in);" ^
  "$utf8NoBom = New-Object System.Text.UTF8Encoding($false);" ^
  "$sw = [System.IO.StreamWriter]::new($out,$false,$utf8NoBom);" ^
  "$sw.WriteLine('Date,Time,Open,High,Low,Close,TickVolume,Volume');" ^
  "while(($line=$sr.ReadLine()) -ne $null) {" ^
  "  if([string]::IsNullOrWhiteSpace($line)) { continue }" ^
  "  $p = $line.Split(',');" ^
  "  if($p.Length -lt 6) { continue }" ^
  "  try { $t = [int64]$p[0] } catch { continue }" ^
  "  if($t -ge 100000000000000) { $ms = [int64]($t/1000) } else { $ms = $t }" ^
  "  $dt = [DateTimeOffset]::FromUnixTimeMilliseconds($ms).UtcDateTime;" ^
  "  $date = $dt.ToString('yyyy.MM.dd');" ^
  "  $time = $dt.ToString('HH:mm:ss');" ^
  "  $tv = 1;" ^
  "  $sw.WriteLine(('{0},{1},{2},{3},{4},{5},{6},{7}' -f $date,$time,$p[1],$p[2],$p[3],$p[4],$tv,$p[5]));" ^
  "}" ^
  "$sw.Flush(); $sw.Close(); $sr.Close();"

if errorlevel 1 (
  echo ERROR: Conversion failed.
  exit /b 1
)

echo OK: Created "%OUT%"
endlocal
```

## Приложение: convert_ticks_2permin.bat

Назначение: из Binance M1 klines создать синтетические MT5 тики — 2 тика на минуту.

```bat
@echo off
setlocal EnableExtensions

set "IN=BTCUSDT_1y_raw.csv"
set "OUT=BTCUSDT_ticks_2permin.csv"

powershell -NoProfile -ExecutionPolicy Bypass -Command ^
  "$in='%IN%'; $out='%OUT%';" ^
  "$sr=[IO.StreamReader]::new($in);" ^
  "$enc=New-Object Text.UTF8Encoding($false);" ^
  "$sw=[IO.StreamWriter]::new($out,$false,$enc);" ^
  "$sw.WriteLine('Date,Time,Bid,Ask,Last,Volume,Flags');" ^
  "while(($line=$sr.ReadLine()) -ne $null){" ^
  "  if([string]::IsNullOrWhiteSpace($line)){continue}" ^
  "  $p=$line.Split(','); if($p.Length -lt 6){continue}" ^
  "  try{$t=[int64]$p[0]}catch{continue}" ^
  "  if($t -ge 100000000000000){$ms=[int64]($t/1000)} else {$ms=$t}" ^
  "  $dt=[DateTimeOffset]::FromUnixTimeMilliseconds($ms).UtcDateTime;" ^
  "  $date=$dt.ToString('yyyy.MM.dd');" ^
  "  $t0=$dt.ToString('HH:mm:ss.000');" ^
  "  $t1=$dt.AddMilliseconds(59999).ToString('HH:mm:ss.999');" ^
  "  $open=$p[1]; $close=$p[4];" ^
  "  $sw.WriteLine(('{0},{1},{2},{2},{2},1,30' -f $date,$t0,$open));" ^
  "  $sw.WriteLine(('{0},{1},{2},{2},{2},1,30' -f $date,$t1,$close));" ^
  "}" ^
  "$sw.Close(); $sr.Close();"

echo OK: %OUT%
endlocal
```
