## Краткое резюме

В чате анализировались фильтры запрета старта сетки для FLAT-робота на MT5. Основные параметры фильтрации: VI, ADX и SmartMode. Проверялись варианты без умного режима, с умным режимом 0.01 и 0.02, а также одиночные и комбинированные блоки защиты старта.

Главная цель: не просто улучшить PnL одного месяца, а найти фильтр, который стабильно улучшает распределение результатов по нескольким месяцам и помогает удерживать работу внутри сетки. Приоритетная цель стратегии — увеличивать долю V6 и снижать ущерб V1–V4.

## Идеи

- Фильтр старта не должен оцениваться по одному месяцу. Нужна проверка минимум на февраль, март и апрель.
- Фильтр старта не может напрямую знать будущий исход V1–V6. Он только меняет вероятность входа в определённых рыночных режимах.
- Улучшение PnL даже без роста V6 может быть полезным, если оно устойчиво и предсказуемо снижает убыток.
- На малой сетке с 2 шагами получить сильный рост V6 сложно. Поэтому любые стабильные улучшения распределения имеют значение.
- Цель умного режима — не «запретить все плохие события», а сместить работу в сторону более безопасных и прибыльных сценариев.

## Гипотезы

### Гипотеза 1: блоки 2 и 3 после сужения могут быть полезны

Старые блоки 2 и 3 ухудшали или почти не улучшали результат. После замены на более узкие варианты влияние стало лучше в феврале и апреле.

Проверка:
- сравнить старые и новые блоки 2/3 по всем месяцам;
- смотреть не только лучший PnL, а изменение относительно базового режима;
- отдельно проверить, не является ли улучшение случайным эффектом одного периода.

### Гипотеза 2: комбинации 2+4 и 3+4 выглядят перспективнее одиночных блоков

Комбинации 2+4 и 3+4 дали сильное улучшение в апреле и феврале. В марте почти не изменили картину. Это лучше, чем резкое ухудшение.

Проверка:
- прогнать 2+4 и 3+4 на дополнительных месяцах;
- сравнить средний PnL, худший месяц и стабильность;
- проверить распределение V1–V6, а не только общий PnL.

### Гипотеза 3: март требует отдельной диагностики

В марте почти все фильтры дают слабое изменение. Значит, март может быть режимом, где проблема не в старте, а в самой конфигурации сетки, шаге, буфере, трейлинге или структуре выхода.

Проверка:
- разобрать март по сессиям;
- посмотреть, какие VI/ADX реально дают основные убытки;
- проверить, не нужны ли другие параметры сетки вместо фильтра старта.

## Выводы

### 1. Старые ADX-фильтры были слабыми

Старые варианты блоков 2 и 3 по ADX выглядели плохо. В апреле они ухудшали результат, особенно блок 3. Простая идея «запретим ADX-зону» оказалась слишком грубой. Индикатор опять сделал вид, что он мозг. Как обычно, рынку всё равно.

### 2. Блок 1 неоднозначен

Блок 1 иногда улучшает результат в сочетании с SmartMode, но сам по себе не универсален.

Примеры:
- февраль: блок 1 хуже базы;
- март: блок 1 без SmartMode сильно хуже, но с SmartMode 0.01 лучше базы;
- апрель: блок 1 с SmartMode 0.02 дал хороший результат.

Вывод: блок 1 нельзя считать самостоятельным универсальным фильтром без проверки комбинаций.

### 3. Блок 4 устойчиво интересный

Блок 4 часто даёт улучшение или близкий к базе результат.

По новым таблицам:
- февраль: блок 4 лучший или один из лучших;
- март: почти без изменений;
- апрель: блок 4 лучше базы без SmartMode и с 0.01, но хуже с 0.02.

Вывод: блок 4 стоит оставить в пуле кандидатов.

### 4. Комбинации 2+4 и 3+4 выглядят сильнее одиночных блоков

По апрелю:
- без защиты: -855 / -866 / -743
- 2+4: -616 / -619 / -721
- 3+4: -616 / -613 / -708

По февралю:
- без защиты: -232 / -178 / -297
- 2+4: -92 / -21 / -138
- 3+4: -91 / -19 / -132

По марту:
- без защиты: -1048 / -1536 / -1070
- 2+4: -1047 / -1029 / -1071
- 3+4: -1049 / -1031 / -1073

Вывод: 2+4 и 3+4 не решают март, но улучшают февраль и апрель. Это достаточная причина оставить их для дальнейшей проверки.

### 5. Выбор фильтра должен быть по нескольким месяцам

Нельзя вычеркивать фильтр по одному месяцу. Нужно смотреть общую устойчивость:
- средний PnL;
- худший месяц;
- количество месяцев с улучшением;
- сила ухудшения в слабых месяцах;
- влияние на V1–V6.

## Ошибки и решения

### Ошибка: попытка привязать фильтр старта к будущему исходу V1

Фильтр старта видит только текущий режим рынка. Он не знает, будет ли будущая сессия V1, V2 или V6.

Правильная логика:
- фильтр оценивается только статистически;
- сравнивается распределение исходов после большого числа стартов;
- вывод строится по вероятностному сдвигу, а не по отдельным сделкам.

### Ошибка: оценка только по лучшему PnL

Оценка только по лучшему значению искажает вывод. Нужно смотреть влияние фильтра на всю матрицу.

Правильная логика:
- сравнивать каждую конфигурацию с базой того же месяца;
- учитывать, где фильтр улучшил, ухудшил или почти ничего не изменил;
- отдельно смотреть worst-case.

### Ошибка: старые блоки 2 и 3 были слишком грубыми

Старые блоки:
- блок 2: VI 1.0–1.5 + ADX < 15;
- блок 3: ADX 18–28.

Они ухудшали результат в апреле и почти не давали стабильной пользы.

Решение:
- заменить на более узкие варианты;
- не выкидывать идею полностью до проверки по нескольким месяцам.

## Правила

### Правило 1: фильтр старта оценивается по распределению

Фильтр не «ловит» конкретный будущий V1. Он меняет вероятность попадания в разные сценарии.

Оценивать:
- totalPnL;
- ExitPNL;
- V1–V6;
- изменение количества сессий;
- стабильность по месяцам.

### Правило 2: PnL можно использовать для ускоренной первичной проверки

На раннем этапе допустимо смотреть только PnL, чтобы быстро отсеять явный мусор.

Но финальное решение нельзя принимать только по PnL. Нужна проверка:
- V1/V2;
- V5/V6;
- число сессий;
- распределение по режимам VI/ADX.

### Правило 3: лучший фильтр выбирается по всем месяцам

Минимум:
- февраль;
- март;
- апрель.

Лучше:
- добавить май, июнь, июль;
- отдельно проверить кризисные и спокойные участки.

### Правило 4: если месяц не реагирует на фильтры старта, искать причину глубже

Если все фильтры почти не меняют результат, проблема может быть не во входе. Возможные причины:
- шаг сетки;
- ширина сетки;
- буфер;
- трейлинг;
- логика выхода;
- рыночный режим после старта.

## Архитектура

### Текущая логика SmartADX

В коде SmartADX используется как режим выбора фильтра:

```mql5
input int SmartADX = 1; // Smart: фильтрация ADX: 0=выкл, 1-4 блоки, 5=все, 6=2+4, 7=3+4
```

Рабочая идея:
- `0` — без защиты старта;
- `1` — блок 1;
- `2` — блок 2;
- `3` — блок 3;
- `4` — блок 4;
- `5` — все блоки;
- `6` — блоки 2+4;
- `7` — блоки 3+4.

### Regime_BeginSession

Функция фиксирует режим рынка на старте сетки:
- `VI = ATR100 / ATR1000`;
- `ADX50`;
- `AbsVol`;
- `gridStepPerc_smart`;
- коэффициенты шага, буфера, трейлинга.

Принцип:
- параметры фиксируются на старте сессии;
- фильтр старта возвращает `true`, если запуск запрещён;
- после завершения сессии результат записывается в статистику.

### Статистика V1–V6

Используются классы завершения:
- V1 — чистый стоп;
- V2 — была защита, но без тейка;
- V3/V4 — защита плюс тейки;
- V5 — форс-мажор;
- V6 — выход по take-limit / работа внутри сетки.

Цель:
- меньше V1–V4;
- больше V6;
- меньше убыток при плохих сценариях.

### Главный принцип анализа

Фильтры старта не влияют напрямую на результат. Они меняют вероятность попадания в рыночные режимы. Поэтому важнее не один лучший PnL, а распределение исходов по месяцам и V1–V6.
