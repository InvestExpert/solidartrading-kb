# AI-инфраструктура TiTan Bots - минимальная сложность

## Оценка ценности

1/10.

Статус: `archive_processed / absorbed`.

Карточка закрыта как слишком общая. Она не дает отдельного внедряемого действия, а полезный остаток уже покрыт текущей архитектурой базы и отдельными рабочими карточками.

## Решение

Не оставлять как активную идею.

Смысл карточки был правильный, но слишком общий:

```text
Не усложнять AI-инфраструктуру без конкретной задачи.
```

Этот принцип уже покрыт в базе:

- `Система/Skills/titan-bots-wiki-import/SKILL.md` - правило `Keep the system simple`;
- `Система/Правила обработки/Регламент регулярных автоматизаций базы.md` - границы регулярной автоматизации;
- `Codex workflow для TiTan Bots - идеи` - закрытая сборная карточка; выбор Codex workflow / MCP / subagents / automations теперь держится в конкретных правилах и future-идеях;
- `К разбору/Идеи/Claude Code skills и аналоги для Codex workflow.md` - карта Codex-поверхностей;
- активные FAQ/METHOD/RAG-правила - уже описывают AI-помощника и route/RAG слой.

## Что не выделять отдельно

- `RAG-база / Telegram-ассистент` - уже есть активный AI-помощник и FAQ/RAG маршруты.
- `Агентный аудит багов` - частично покрыт audit-stack, SYNTHESIS, Codex workflow и ручными очередями `К разбору/Ошибки` / `К разбору/Идеи`.
- `MCP только под конкретную внешнюю систему` - это не самостоятельная задача без выбранной системы и действия.
- `Субагенты только для независимых частей` - уже покрыто Codex workflow / Superpowers-подходом.
- `Аналитика контента по деньгам, а не просмотрам` - не относится к AI-инфраструктуре и требует отдельного маркетингового решения, если понадобится.

## Что осталось полезным

Практический смысл сохранен как принцип:

- сначала простое решение;
- skill или правило только для повторяемой задачи;
- RAG/Telegram только поверх чистой wiki;
- агент или subagent только там, где есть цепочка действий или независимые части;
- MCP только под конкретную внешнюю систему/API и конкретное действие;
- автономные действия без review не внедрять.

## Почему закрыто

Карточка не нужна как отдельная active idea, потому что смешивает RAG, MCP, agents, subagents, Telegram-бота и аналитику без одного конкретного next action.

Конкретные новые задачи лучше фиксировать отдельными карточками. Пример уже создан: `SolidarTrading - AI-агенты для управления балансом мастер-счета`.
