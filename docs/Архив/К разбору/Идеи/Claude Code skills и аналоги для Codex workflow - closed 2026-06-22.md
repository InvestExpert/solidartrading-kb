# Claude Code skills и аналоги для Codex workflow

## Оценка ценности

`8/10`

Идея ценная, потому что источник перечисляет не абстрактные "AI skills", а конкретные классы задач: свежая документация, экономия токенов через маршрутизацию моделей, превращение размытой задачи в ТЗ, разбиение большой фичи на шаги и браузерное тестирование. Оставить в инкубаторе, потому что инструменты из видео ориентированы на Claude Code, а для Codex нужно проверять аналоги отдельно.

## Идея

Проверить 5 классов инструментов из видео про Claude Code и подобрать для них безопасные аналоги в Codex workflow TiTan Bots.

Не ставить инструменты из видео автоматически. Сначала понять, какую задачу они решают, есть ли уже встроенный или официальный аналог в Codex, и нужна ли отдельная TiTan Bots-адаптация.

## Кандидаты и аналоги для Codex

### 1. Context7 MCP для свежей документации

Ценность: `8/10`.

Задача из видео:

- дать агенту актуальную документацию по стеку;
- снизить риск, что агент пишет код по устаревшим знаниям.

Возможный аналог для Codex:

- Codex поддерживает MCP;
- Context7 можно рассматривать как MCP-кандидат для документации;
- для OpenAI/Codex использовать официальные OpenAI docs и manual, а не сторонний MCP.

Что проверить:

- нужен ли Context7 для реального стека TiTan Bots;
- какие языки и библиотеки реально требуют свежей документации;
- не достаточно ли официальных docs, локальной wiki и targeted web lookup.

### 2. Claude Code Router для выбора модели и экономии токенов

Ценность: `6/10`.

Задача из видео:

- маршрутизировать простые задачи на более дешевые или локальные модели;
- не тратить дорогую модель на мелкие правки.

Возможный аналог для Codex:

- ручной выбор модели через настройки Codex, `/model`, CLI flags или provider/API config;
- отдельная проверка локальных/внешних провайдеров только если есть реальная экономия без потери качества.

Что проверить:

- есть ли у TiTan Bots достаточно легких повторяемых задач, где дешевый режим безопасен;
- можно ли разделить задачи на cheap/normal/high-risk без ошибок;
- не опасно ли отдавать trading/wiki задачи слабой модели.

### 3. Spec Kit как перевод размытой задачи в ТЗ

Ценность: `8/10`.

Задача из видео:

- превратить поток мыслей в конкретное техническое задание;
- сначала согласовать план, потом писать код.

Возможный аналог для Codex:

- Codex skill для preflight/TЗ;
- `AGENTS.md` или skill-правило "размытая задача -> план -> approval -> минимальная правка";
- использовать существующий approval-gate для wiki/import и расширить только при повторяющейся пользе.

Что проверить:

- взять 2-3 реальные размытые задачи;
- сравнить результат без отдельного Spec-style прохода и с ним;
- не добавляет ли это лишний ритуал для маленьких задач.

### 4. Task Master для больших фич и пошагового ведения

Ценность: `7/10`.

Задача из видео:

- разбивать большую фичу на цепочку шагов;
- вести агента по плану, чтобы он не терял контекст и не ломал проект.

Возможный аналог для Codex:

- plan-first workflow;
- subagents для независимых частей;
- worktrees для параллельных задач;
- checklist в плане или отдельная task-card только для крупных работ.

Что проверить:

- какие TiTan Bots задачи реально требуют 20-30 шагов;
- нужен ли внешний Task Master или достаточно плана в Codex;
- как не превратить маленькие правки в тяжелый process overhead.

### 5. Playwright / browser testing

Ценность: `7/10`.

Задача из видео:

- заставить агента проверять UI как пользователь;
- кликать, открывать страницы, смотреть результат.

Возможный аналог для Codex:

- Codex in-app browser / Browser Use;
- Playwright или MCP только если нужен отдельный автоматизированный browser layer;
- для текущих wiki/import задач это не приоритет, но для будущих Telegram WebApp, лендингов, кабинетов и dashboard может быть важно.

Что проверить:

- есть ли ближайший UI, который нужно тестировать агентом;
- хватает ли in-app browser;
- когда нужен именно Playwright test вместо визуальной проверки.

## Почему может быть ценно

- помогает отделить реальные улучшения workflow от хайпа вокруг Claude Code;
- дает карту "задача -> Codex surface": MCP, skill, AGENTS.md, model config, browser, subagents, worktrees;
- может усилить будущие Codex skills для TiTan Bots;
- снижает риск ставить чужие инструменты без понимания, что они решают.

## К чему относится

- Codex workflow;
- TiTan Bots wiki imports;
- будущие skills;
- работа с большими задачами;
- browser/UI testing;
- экономия токенов и выбор модели.

## Что проверить

1. Context7 MCP только на реальной задаче со свежей документацией.
2. Spec-style preflight для размытой задачи перед кодом.
3. Task Master-like decomposition для одной большой фичи.
4. Browser Use / Playwright на будущем UI.
5. Модельный router только после явного списка безопасных low-risk задач.

## Как понять, что идея сработала

- Codex меньше пишет устаревший код из-за нехватки документации;
- размытые задачи превращаются в понятный план без лишних переписок;
- большие фичи дробятся на проверяемые шаги;
- UI-проверки находят реальные ошибки;
- экономия токенов не ухудшает качество и безопасность.

## Риски или сомнения

- Видео ориентировано на Claude Code, не на Codex.
- Названия и команды из транскрипта могут быть искажены.
- GitHub stars и рекламные заявления автора не считать доказательством качества.
- Установка сторонних MCP/router/tools может создать риск безопасности.
- Слишком много инструментов может усложнить процесс сильнее, чем помочь.
