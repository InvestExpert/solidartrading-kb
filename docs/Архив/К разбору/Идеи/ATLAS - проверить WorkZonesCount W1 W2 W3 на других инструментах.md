# ATLAS - проверить WorkZonesCount W1 W2 W3 на других инструментах

## Оценка ценности

1/10.

Статус: `archive_processed / obsolete`.

Карточка устарела: это продолжение старой тестовой ветки. Полноценные тесты по V2 и V3 уже сделаны на других инструментах; V2 закреплен как стандарт, V3 прорабатывается отдельно как альтернатива с меньшей просадкой.

Отдельная старая идея по проверке `W1/W2/W3` больше не нужна. Новую задачу не создавать.

## Идея

Проверить `ProgLot_WorkZonesCount = 1/2/3` на других инструментах и периодах, потому что BTC-тест показал сильное влияние `W`, но не доказывает универсальность.

## Решение владельца

Закрыть как устаревшую старую тестовую ветку.

Актуальная логика V2/V3 уже отражена в свежих материалах: V2 принят как стандарт, V3 идет отдельной альтернативой с меньшей просадкой. Эту карточку не держать в активной очереди и не превращать в новую задачу.

## Почему может быть ценно

- `W=1` может быть базовым универсальным стандартом.
- `W=2` может быть отдельным profit-mode.
- `W=3` может быть агрессивным режимом для отдельных инструментов.
- Проверка поможет не принять локальный пик оптимизатора за устойчивую настройку.

## К чему относится

- [TiTan Atlas (BA)](../../../../TiTan%20Atlas%20(BA).md)
- MT5
- Progressive Zoning
- `ProgLot_WorkZonesCount`
- `ProgLot_ZoneWidthSteps`
- оптимизация параметров

## Что проверить

- Прогнать `W=1/2/3` на других инструментах.
- Проверить те же значения на других периодах BTC.
- Отдельно сравнить спокойные и кризисные участки.
- Для универсальной конфигурации сравнивать:
  - число живых проходов `DD < 40%`;
  - число живых проходов `DD < 50%`;
  - лучший profit при `DD < 40%`;
  - лучший profit при `DD < 50%`;
  - RF;
  - количество провалов с DD около 100%.
- Не тратить базовый тестовый пул на `W=4-5`, пока нет отдельной причины.

## Как понять, что идея сработала

- Есть повторяемая картина по нескольким инструментам, а не только по BTC.
- Понятно, где `W=1` база, где `W=2` profit-mode, а где `W=3` слишком агрессивен.
- Вывод можно перенести из К разбору в технический слой без оговорки "проверено только на BTC".

## Риски или сомнения

- BTC может вести себя иначе, чем Forex-инструменты.
- На отдельных активах `W=2` или `W=3` могут выглядеть сильнее, но с узким живым коридором.
- Без одинаковой геометрии сравнение будет некорректным.
