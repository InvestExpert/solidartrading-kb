# Skills из видео для проверки

## Раздел

## Источник

Главный источник:

- Видео: **Полный гайд по AI-скиллам: что это, как создать скилл для ИИ-агента и 10 готовых skills**

## Оценка ценности

- Ценность всей заметки: `10/10`.

Почему ценная:

- здесь не общая идея "использовать skills", а список конкретных кандидатов;
- часть кандидатов напрямую влияет на качество кода торговых ботов;
- можно начать проверку с самых ценных пунктов;
- помогает не ставить случайные сторонние skills без отбора.

Почему оставить в К разбору сейчас:

- это список кандидатов, а не утвержденная библиотека skills;
- часть идей уже частично реализована;
- каждый skill надо проверять отдельно на реальной задаче.

## Уже реализовано

Общая идея использовать skills уже реализована.

Сейчас в работе уже есть:

- `titan-bots-wiki-import`;
- `brainstorming`;
- `skill-creator`.

Поэтому эта заметка не про то, что skills вообще нужны. Она про то, какие следующие skills стоит проверить.

## Кандидаты

### 1. Скилл аудита кода робота

Ценность: `10/10`.

Суть:

Отдельный skill для строгого аудита кода торгового бота.

Что должен проверять:

- входы функции;
- выходы функции;
- вызовы соседних функций;
- глобальные флаги;
- recovery;
- совместимость с соседними блоками;
- риск поломки торговой логики;
- минимальность правки.

Почему ценный:

- напрямую снижает риск сломать торгового бота;
- подходит для MT5, Python и будущих платформ;
- может стать первым кандидатом на реальный тест.

Что проверить:

- взять одну реальную правку бота;
- сравнить review без skill и review со skill;
- проверить, уменьшает ли skill лишние правки и пропущенные риски.

### 2. MT5 vs Python parity

Ценность: `9/10`.

Суть:

Skill для сравнения поведения MT5 и Python-версий там, где логика должна совпадать.

Что сравнивать:

- число сделок;
- BUY/SELL;
- forced;
- средний profit ordinary SELL;
- средний forced loss;
- winrate;
- глубины;
- live_buy_count;
- повторяющиеся последовательности событий.

Почему ценный:

- важен для переноса логики между платформами;
- помогает искать не только строковые расхождения, а поведенческие.

Что проверить:

- выбрать один бот с MT5/Python-версией;
- подготовить логи;
- проверить, может ли skill найти реальные расхождения.

### 3. Анализ оптимизатора

Ценность: `9/10`.

Суть:

Skill для разбора результатов MT5 optimizer.

Что должен делать:

- не выбирать одиночный лучший результат;
- искать плато;
- выделять локальные пики;
- учитывать DD;
- учитывать profit;
- смотреть worst-case;
- формировать shortlist;
- отделять универсальные параметры от инструментальных.

Почему ценный:

- помогает не переобучаться на красивый пик;
- полезен для поиска устойчивых настроек.

Что проверить:

- взять реальную таблицу оптимизатора;
- сравнить вывод skill с ручным анализом.

### 4. Выжимка чатов для Obsidian

Ценность: `8/10`.

Суть:

Skill для превращения длинных чатов в структурированные материалы для wiki.

Статус:

Частично реализовано через `titan-bots-wiki-import`.

Что проверить:

- какие задачи уже закрывает текущий skill;
- нужен ли отдельный `titan-bots-video-extract`;
- нужен ли отдельный skill для ChatGPT-выжимок.

### 5. Security audit сторонних skills

Ценность: `8/10`.

Суть:

Проверка сторонних skills перед установкой или копированием.

Что проверять:

- скрытые инструкции;
- попытки доступа к секретам;
- попытки отправить данные наружу;
- команды на чтение файлов;
- команды на поиск ключей, токенов и паролей;
- команды, которые могут изменить проект без разрешения.

Почему ценный:

- снижает риск поставить чужую опасную инструкцию;
- особенно важен, если дальше искать готовые skills.

Что проверить:

- взять 2-3 сторонних skill;
- прогнать ручной аудит;
- понять, нужен ли отдельный skill для проверки.

### 6. Документация в skill

Ценность: `7/10`.

Суть:

Превращать документацию платформы, API или инструмента в короткий рабочий skill.

Где может пригодиться:

- MT5;
- MQL5;
- Python;
- Backtrader;
- CCXT;
- Telegram bot platforms;
- RAG-сервисы;
- Obsidian workflow.

Что проверить:

- взять одну документацию;
- сделать черновой skill;
- проверить на реальной задаче.

### 7. AI/RAG бот по проекту

Ценность: `7/10`.

Суть:

Будущий ассистент, который отвечает по базе TiTan Bots.

Что может делать:

- отвечать по материалам TiTan Bots;
- объяснять метод;
- находить презентации, инструкции и FAQ;
- поддерживать партнерские ссылки;
- отвечать обычным языком без ручного меню.

Почему пока К разбору:

- база еще чистится;
- нужно четко отделить wiki от сырых и рискованных идей;
- рано делать ассистента до стабилизации структуры.

### 8. Отчет по версии робота

Ценность: `7/10`.

Суть:

Skill для подготовки отчетов по версиям, тестам, скринам и изменениям.

Почему ценный:

- полезен для истории версий;
- может помогать готовить материалы для презентаций и внутренних выводов;
- снижает хаос после изменений кода.

Что проверить:

- взять одну версию бота;
- собрать по ней короткий отчет;
- оценить, хватает ли данных в wiki.

### 9. Диаграммы из текста

Ценность: `6/10`.

Суть:

Skill для превращения описаний архитектуры в схемы.

Где может пригодиться:

- NORMAL / FORCE / SUPER;
- RiskGuard;
- ConnectionSupervisor;

- pipeline тестов;
- презентационные схемы.

Ограничение:

Диаграмма не заменяет техническую спецификацию.

### 10. Редактор без воды

Ценность: `5/10`.

Суть:

Skill для чистки текстов постов, отчетов и описаний.

Почему не выше:

- может быть полезен;
- но может ослабить продающий стиль;
- для продажных материалов нужен отдельный контроль языка и выгоды.

### 11. Research последних 30 дней

Ценность: `5/10`.

Суть:

Skill для поиска свежих обсуждений по AI-агентам, trading bots, RAG и Telegram bots.

Почему осторожно:

- может дать идеи;
- но соцсети и новости дают много шума;
- требует проверки источников.

### 12. Marketing skills

Ценность: `5/10`.

Суть:

Готовые marketing skills для лендингов, писем, воронок и контент-планов.

Почему осторожно:

- могут дать шаблонный текст;
- могут ослабить простую человеческую подачу;
- продажные материалы нельзя автоматически переносить в `Метод и продвижение`.

### 13. Frontend / slides

Ценность: `4/10`.

Суть:

Skills для быстрой упаковки презентаций, лендингов и визуальных материалов.

Почему не приоритет:

- полезно для визуала;
- но вторично относительно разработки, аудита и оптимизации ботов.

### 14. Перевод больших документов

Ценность: `3/10`.

Суть:

Skill для перевода больших документов, PDF, книг или документации.

Почему низко:

- иногда полезно;
- но сейчас не ключевая повторяющаяся задача TiTan Bots.

### 15. Поискать еще готовые skills

Ценность: `7/10`.

Суть:

Отдельно поискать готовые skills, которые можно адаптировать под TiTan Bots.

Что искать:

- code review;
- trading logic review;
- optimizer/table analysis;
- Obsidian/wiki extraction;
- RAG assistant;
- security audit.

Почему оставить в К разбору:

- сначала надо понять, какие задачи реально повторяются;
- чужие skills нельзя ставить без аудита;
- готовые skills почти всегда надо адаптировать.

### 16. UI-kit / shadcn-like skill

Ценность: `8/10`.

Суть:

Skill, который заставляет агента использовать готовые компоненты проекта, а не выдумывать интерфейс заново.

Где может пригодиться:

- Telegram WebApp;
- кабинет клиента;
- интерфейс RAG-ассистента;
- страницы отчетов;
- внутренние панели.

Что проверить:

- есть ли у TiTan Bots будущий единый UI-kit;
- нужен ли сначала лендинг, кабинет или Telegram WebApp;
- можно ли использовать готовые компоненты без лишней сложности.

### 17. Canvas Design / SVG / инфографика

Ценность: `8/10`.

Суть:

Skill для создания схем, инфографики, баннеров и визуальных объяснений через код или структуру.

Где может пригодиться:

- отчеты;
- презентации;
- схемы работы бота;
- объяснение risk/recovery;
- рекламные материалы;
- обложки и визуалы для постов.

Ограничение:

Визуал не должен заменять проверенный смысл. Сначала смысл и ограничения, потом упаковка.

### 18. PostgreSQL skill

Ценность: `7/10`.

Суть:

Skill для работы агента с PostgreSQL, если будущий RAG/Telegram-бот будет хранить пользователей, материалы, логи, партнерские ссылки или историю запросов.

Почему не выше:

- пока нет утвержденной базы данных;
- сначала нужно стабилизировать wiki и сценарии ассистента.

Что проверить:

- нужна ли база данных на первом этапе;
- хватит ли файловой wiki;
- какие сущности действительно надо хранить структурно.

### 19. Skills infrastructure / installer

Ценность: `7/10`.

Суть:

Идея единой установки и переноса skills между средами: Codex, Claude, Cursor и локальные правила.

Почему полезно:

- снижает хаос в правилах;
- помогает не копировать skills вручную;
- может пригодиться, если TiTan Bots будет использовать несколько AI-инструментов.

Что проверить:

- какие среды реально используются;
- нужен ли общий формат;
- какие skills должны быть переносимыми, а какие только для Codex.

## Что не переносить

- Рекламные части источника.
- Призывы автора видео.
- Общую идею "использовать skills", потому что она уже внедрена.
- Утверждение "внедрить все 9 Claude Skills".
- Точные названия skills без проверки, потому что транскрипт может искажать названия.
- Продажные формулировки и идеи контента.
- Любые материалы в `Метод и продвижение` без отдельного одобрения.

## Что проверить первым

1. Скилл аудита кода робота.
2. MT5 vs Python parity.
3. Анализ оптимизатора.
4. Security audit сторонних skills.

Именно эти пункты выглядят самыми полезными для TiTan Bots сейчас.
