# Skills - идеи развития TiTan Bots

## Оценка ценности

`9/10`.

Статус: канонический backlog будущих skills TiTan Bots.

Эта карточка объединяет старую карточку `Skills - идеи развития TiTan Bots` и карточку `Skills из видео для проверки`. Второй файл закрыт как `absorbed`, чтобы в активной очереди не было двух конкурирующих списков.

Итог 2026-06-22: карточка разобрана и закрывается как active backlog. Рабочие решения перенесены в `Система/Skills`, `Skills в работе TiTan Bots` и preflight/router. Дальше новые улучшения skills должны попадать в grouped idea `META-SYSTEM - advisory review качества skills`, а не держаться в этой старой сборной карточке.

## Принцип

Новые skills нужны только там, где есть повторяемая задача, риск ошибки или заметная экономия времени.

Не ставить и не создавать skills "потому что они есть в видео". Сначала определить рабочую боль, потом проверить на одной реальной задаче, потом решать, делать ли постоянный skill.

## Уже закрыто или покрыто

Эти пункты не держать как отдельные идеи:

- общая идея использовать skills - уже реализована;
- `titan-bots-wiki-import` - уже рабочий главный skill для базы;
- `copywriter` - уже рабочий skill для текста и продажной подачи;
- Superpowers - уже установлен пакет, правила использования описаны в `Skills в работе TiTan Bots`;
- preflight крупных задач - уже принят как легкий router внутри `titan-bots-wiki-import`;
- жесткая проверка перед завершением wiki-работы - покрыта обязательным FAST;
- общий Codex workflow / AGENTS / plans / checkpoint / automations - закрыт отдельными системными правилами;
- общий MCP без конкретной внешней системы - не самостоятельная активная идея;
- общий AI/RAG-ассистент - не skill-задача сейчас, а будущая продуктовая/RAG-архитектура.

## Кандидаты первого уровня

### 1. `titan-bots-code-review`

Приоритет: `10/10`.

Статус: создан, перенесен в рабочий слой и первично проверен 2026-06-22.

Назначение: строгий статический review правок торгового кода без вольного рефакторинга.

Объединяет старые пункты:

- скилл аудита кода робота;
- review торговой логики;
- аудит риска сеточных ботов.

Что должен проверять:

- входы и выходы функции;
- соседние вызовы и влияние на цепочку торговли;
- глобальные флаги и состояние;
- RiskGuard / forced sell / forced exits;
- recovery и rebuild состояния;
- изменение торгового поведения;
- совместимость с соседними блоками;
- минимальность правки;
- риск, что защита есть в коде, но не сработает в реальном сценарии.
- concurrency chain: двойные входы, гонки, быстрые тики, async/multi-symbol риски;
- side-effects chain: непреднамеренные изменения общих dict/list/global state;
- failure chain: retcode/API timeout/exception, rollback локального состояния, safe fallback;
- freshness chain: устаревшие цены, свечи, balances, last order, heartbeat;
- complexity budget: новые переменные, функции, state keys, зависимости, переименования;
- risk matrix: severity, evidence, required action;
- `chain_map_inferred`, если формальная карта цепочки отсутствует и карта построена по коду.

Почему первый кандидат:

- напрямую снижает риск сломать торгового бота;
- применим к MT5, Python и будущим платформам;
- дает пользу уже на ближайшей серьезной правке кода.

Проверка перед внедрением:

- skill создан в `Система/Skills/titan-bots-code-review`;
- установленная копия синхронизируется в `C:\Users\Oleksiy Titov\.codex\skills\titan-bots-code-review`;
- первичная проверка выполнена на сравнении `BA 18 4 BUY.mq5` -> `BA 18 5 BUY.mq5`;
- дальнейшая доработка идет по реальным code-review случаям, а не как отдельная активная идея.

### 2. `mt5-python-parity-audit`

Приоритет: `9/10`.

Решение 2026-06-22: не делать отдельным skill сейчас. Встроено как режим `parity_translation_review` в `titan-bots-code-review`, потому что это review переноса торговой логики между платформами.

Назначение: сравнивать поведение MT5 и Python-версий, когда логика должна совпадать.

Что сравнивать:

- число сделок;
- BUY/SELL;
- forced / ordinary exits;
- средний profit ordinary SELL;
- средний forced loss;
- winrate;
- глубины;
- live_buy_count;
- повторяющиеся последовательности событий;
- расхождения между кодом и логами.

Почему ценно:

- важно для переноса логики между MT5 и Python;
- ищет не только строковые различия, а поведение стратегии;
- особенно полезно для ATLAS Crypto / Spot / Python-переносов.

Проверка перед внедрением:

- выбрать один бот с MT5/Python-версией;
- подготовить сопоставимые логи или отчеты;
- проверить, может ли skill найти реальные расхождения.

### 3. `optimizer-result-analyzer`

Приоритет: `9/10`.

Назначение: разбирать результаты MT5 optimizer и не выбирать параметры по одному красивому пику.

Что должен делать:

- искать плато устойчивых параметров;
- отделять случайный пик от рабочей зоны;
- учитывать profit, DD, worst-case, количество сделок и стабильность по периодам;
- формировать shortlist кандидатов;
- отделять универсальные параметры от инструментальных;
- показывать, какие параметры требуют перепроверки.

Почему ценно:

- снижает риск переобучения;
- напрямую связано с текущими тестами и оптимизациями;
- может улучшить качество выбора конфигураций.

Проверка перед внедрением:

- взять реальную таблицу оптимизатора;
- сравнить вывод skill с ручным анализом;
- проверить, не дает ли skill красивую, но бесполезную аналитику.

## Кандидаты второго уровня

### 4. `third-party-skill-security-audit`

Приоритет: `8/10`.

Назначение: проверять сторонние skills перед установкой или копированием.

Что проверять:

- скрытые инструкции;
- попытки доступа к секретам;
- попытки отправить данные наружу;
- команды на чтение ключей, токенов и паролей;
- команды, которые могут менять проект без разрешения;
- установочные действия, которые создают неожиданный footprint.

Решение сейчас:

- отдельный skill не создавать;
- минимальное правило добавлено в `Skills в работе TiTan Bots` и preflight/router: чужие skills не ставить без review содержимого, scripts, commands, external access, secrets risk и footprint.

### 5. `titan-bots-code-version-report`

Приоритет: `7/10`.

Назначение: готовить отчет по версии робота после изменений кода, тестов и скринов.

Что должен давать:

- что изменилось;
- какие версии/файлы затронуты;
- какие тесты или отчеты связаны;
- какие выводы можно переносить в wiki;
- какие риски остались.

Статус: полезно позже, после стабилизации code-review процесса.

### 6. `titan-bots-video-extract`

Приоритет: `7/10`.

Назначение: разбирать видео, конспекты и длинные выжимки в идеи/правила/материалы TiTan Bots.

Что уже покрыто:

- базовая обработка закрывается `titan-bots-wiki-import`;
- YouTube-процесс уже существует.

Что может остаться:

- отдельный слой для массовой обработки видео, если такие импорты станут регулярными.

Статус: отдельный skill сейчас не создавать. Базовый процесс уже покрыт `titan-bots-wiki-import` и YouTube/import правилами. Если появится регулярная массовая обработка видео, META-advisory должен создать новую grouped idea.

### 7. `documentation-to-skill`

Приоритет: `6/10`.

Назначение: превращать документацию платформы/API в короткий рабочий skill или reference.

Где может пригодиться:

- MT5/MQL5;
- Backtrader;
- CCXT;
- Telegram bot APIs;
- RAG-сервисы;
- брокерские API.

Статус: отдельный общий skill не создавать. Процесс закрывается связкой `titan-bots-wiki-import` + `skill-creator` / `writing-skills`: документация сначала разбирается как источник, затем из нее делается конкретный reference или skill под задачу. Общий "документ в skill" без конкретного API/платформы не нужен.

### 8. `titan-bots-optimizer-review`

Приоритет: `10/10`.

Статус: создан и перенесен в рабочий слой 2026-06-22.

Назначение: анализировать ежедневные таблицы оптимизации, MT5 optimizer results, Python/backtest tables, CSV/скрины и ChatGPT-выжимки по подбору параметров.

Решение: это не future. Это один из основных текущих рабочих процессов, поэтому создан отдельный skill `titan-bots-optimizer-review`, добавлен в `Skills в работе TiTan Bots` и router/preflight.

Объединяет:

- UI-kit / shadcn-like skill;
- Claude Frontend Design;
- Vercel Web Design;
- React / Next.js Best Practices;
- frontend/slides.

Приоритет: `4/10`.

Объединяет:

- диаграммы из текста;
- Canvas / SVG / инфографика;

Почему отложено:

Использовать только если будет утверждена архитектура RAG/Telegram-бота с базой данных.

Пока не создавать: файловая wiki и текущие RAG-сценарии важнее.

### MCP Builder

Приоритет: `3/10`.

Не создавать общий MCP-skill без конкретной внешней системы/API.

### Research последних 30 дней

Приоритет: `3/10`.

Не делать отдельным skill сейчас: новости и соцсети дают много шума, а актуальные факты уже должны идти через RESEARCH-процесс.

### Перевод больших документов

Приоритет: `2/10`.

Не является текущей повторяемой задачей TiTan Bots. Использовать обычные document/pdf-инструменты по необходимости.

## Закрыто без отдельного действия

- `titan-bots-trading-code-review` как отдельное название - закрыто переименованием в `titan-bots-code-review`;
- `Редактор без воды` - покрыто `copywriter`, но для продажных материалов нужен approval и сохранение выгоды.
- `Marketing skills` - покрыто `copywriter` и promotion-import; готовые маркетинговые skills могут дать шаблонный текст.
- `Поискать еще готовые skills` - не самостоятельная задача. Сначала выбирается рабочая боль, потом ищется/создается skill.
- `Skills infrastructure / installer` - преждевременно; сейчас достаточно двух копий активных TiTan skills и явной синхронизации.
- `AI/RAG бот по проекту` - это продуктовая/RAG-архитектура, а не один skill.

## Что решать первым

Решено 2026-06-22:

1. `titan-bots-code-review` внедрен и первично проверен.
2. `mt5-python-parity-audit` встроен как режим `titan-bots-code-review`, не отдельный skill.
3. `optimizer-result-analyzer` внедрен как отдельный `titan-bots-optimizer-review`.
4. Минимальное правило security-audit для чужих skills добавлено в активные правила.
5. `titan-bots-video-extract` и `documentation-to-skill` не создаются как отдельные skills сейчас, потому что покрываются существующим import + skill-creator процессом.

## Критерий внедрения

Новый skill внедряется только если:

- есть реальная повторяемая задача;
- есть пример входа и ожидаемый выход;
- skill уменьшает ошибки или время;
- не дублирует уже рабочий `titan-bots-wiki-import`, `copywriter` или Superpowers;
- есть понятная проверка результата;
- есть решение, где хранится skill и как синхронизируется с Codex.

## Что проверить

Карточка не требует дальнейшего активного разбора. Новые предложения по skills должны попадать в `META-SYSTEM - advisory review качества skills` или в отдельную конкретную идею, если есть новая рабочая боль.
