# TiTan Atlas (BA) - crypto hedge строится слоями

## Идея

Для [TiTan Atlas (BA)](../../../../TiTan%20Atlas%20(BA).md) в crypto spot нельзя просто скопировать forex hedge и заменить валютные пары на набор крипто-пар.

В крипте много инструментов могут одновременно зависеть от BTC, ликвидности и общего режима риска. Поэтому защита должна строиться слоями, а не количеством тикеров.

Нельзя считать, что большее количество крипто-пар автоматически снижает портфельный риск. Если инструменты одновременно зависят от BTC, общей ликвидности и режима risk-on / risk-off, расширение списка пар может усилить синхронную просадку.

## Слои портфеля

- USDT-пары, но только после проверки общей просадки.
- BTC-кроссы, например `ETH/BTC`, `BNB/BTC`, `SOL/BTC`.
- Некриптовые spot-кандидаты, если они реально доступны через BingX Spot API.
- Резерв в USDT.
- Фильтр запуска по состоянию BTC.
- Futures hedge overlay только как отдельная будущая архитектура, не внутри текущего spot-контура.

Futures hedge overlay не смешивать с текущей Spot BUY-архитектурой. Это отдельный будущий слой, который потребует отдельной проверки риска, исполнения и ограничений.

## Критерии проверки

- портфельную equity-кривую;
- совпадение просадок по времени;
- корреляцию результатов разных инструментов;
- работу Force / RiskGuard по каждому активу и по портфелю;
- доступность новых символов через BingX Spot API;
- влияние USDT-резерва на выживаемость портфеля;
- помогает ли слой снизить синхронность просадок, а не только добавить прибыльный одиночный инструмент.
