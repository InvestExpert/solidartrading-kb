# TiTan Hyperion (FLAT) - SmartMode v2 symmetric step

## Суть

SmartMode v2 для [TiTan Hyperion (FLAT)](../../../TiTan%20Hyperion%20(FLAT).md) должен быть симметричной системой адаптации шага сетки.

Он не должен только увеличивать шаг. Он должен уметь:

- уменьшать шаг в спокойных mean-reversion режимах;
- увеличивать шаг в опасных режимах;
- сохранять хорошие месяцы;
- стабилизировать плохие месяцы.

## Применимо к

- [TiTan Hyperion (FLAT)](../../../TiTan%20Hyperion%20(FLAT).md)
- MT5 / MQL5
- SmartMode
- adaptive grid step
- V1-V6 statistics

## Источник

`flat_smartmode_v2_plan.md`

## Главная цель

SmartMode v2 должен:

- работать одинаково на разных месяцах;
- не ухудшать хорошие месяцы;
- стабилизировать плохие;
- переводить прибыль в V6;
- минимизировать V1-V4.

Ключевой принцип:

```text
прибыль должна идти через V6, а не через выживание сетки
```

## Почему текущего VI недостаточно

Текущий VI:

```text
ATR(100) / ATR(1000)
```

Он показывает относительную волатильность: ускорение или изменение режима относительно прошлого состояния рынка.

Проблема: рынок может быть тихим относительно себя, но все еще абсолютно волатильным.

То есть `VI = 1.0` может означать:

- нормальный спокойный рынок;
- высокий ATR после длительного тренда, где сужать шаг опасно.

## Два измерения SmartMode

SmartMode v2 должен учитывать две оси:

### Relative VI

Показывает локальное ускорение и изменение режима.

### Absolute ATR regime

Показывает абсолютный масштаб волатильности.

Возможные варианты:

- `ATR(100) / годовой ATR`;
- percentile ATR за год.

Режимы:

- LOW volatility regime;
- NORMAL volatility regime;
- HIGH volatility regime.

## Матрица режимов

| Relative VI | Absolute ATR | Режим |
| --- | --- | --- |
| низкий | низкий | можно сужать |
| низкий | высокий | нельзя сужать |
| высокий | низкий | локальный импульс |
| высокий | высокий | опасность |

## Симметричный SmartMode

План v2:

- 3 ступени вниз;
- 3 ступени вверх.

Это отличается от подхода "только расширять шаг".

Если SmartMode умеет только расширять шаг, он может защищать от опасных режимов, но не умеет зарабатывать на спокойных возвратных режимах.

## Практический вывод

Вероятная форма универсального SmartMode:

- слегка уменьшать шаг в спокойных режимах;
- умеренно увеличивать шаг в опасных режимах;
- иметь отдельный failsafe;
- учитывать абсолютную волатильность.

Иначе система остается слепой к реальному масштабу рынка.

## Что проверить

1. Базу `0.33`.
2. Симметричный SmartMode: 3 ступени вниз и 3 ступени вверх.
3. Отдельно:
   - только step;
   - только buffer;
   - только trailing.
4. Какие параметры реально уменьшают V1.
5. Какие параметры увеличивают V6.
6. Какие параметры просто двигают PnL косметически.
7. Январь, потом февраль, март и multi-month.
