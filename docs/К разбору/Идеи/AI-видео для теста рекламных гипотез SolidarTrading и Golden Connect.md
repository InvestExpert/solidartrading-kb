# AI-видео для теста рекламных гипотез SolidarTrading и Golden Connect

## Оценка ценности

`8/10`

Идея ценная, потому что короткие AI-видео могут быстро проверять рекламные гипотезы без тяжелого продакшена: оффер, метафору, персонажа, визуальную подачу, удержание и переходы. Но источник не подтверждает текущие лимиты, права, доступность и коммерческое использование Google/Veo, поэтому это не готовый процесс и не активное правило.

## Идея

Проверить AI-видео как быстрый формат теста рекламных гипотез:

```text
оффер -> микросюжет -> 8-секундный ролик -> Telegram / Reels / Shorts -> метрики реакции
```

Ролик не должен быть просто "прикольным". Он должен проверять конкретный рекламный тезис: понятность оффера, интерес к теме, доверие к роботу, объяснение механики, желание перейти дальше.

## Почему может быть ценно

- Быстро тестировать разные углы подачи без съемки и монтажа с нуля.
- Делать простые визуальные метафоры для сложных тем: робот, защита, просадка, пассивный режим, статистика, партнерка.
- Подготовить повторяемый визуальный стиль через постоянного персонажа или робота.
- Получать материалы для Telegram, Shorts/Reels, презентаций, сторис и рекламных тестов.
- Проверять реакцию аудитории до создания большого видео, лендинга или полной презентации.

## К чему относится

- SolidarTrading - короткие ролики про робота, защиту, просадку, пассивный режим, статистику и подключение.
- Golden Connect - ролики про рекламную площадку, аудиторию, контакты, кэшбэк, AI-инструменты и пользу для партнера.
- TiTan Bots - визуальное объяснение торговых механизмов простыми сценами.
- Трафик - быстрые креативы для теста удержания, CTR, переходов, заявок и регистраций.

## Возможный pipeline

```text
идея ролика
-> LLM пишет video prompt на английском
-> русские реплики остаются внутри prompt
-> генерация короткого ролика
-> обрезка артефактов
-> публикация теста
-> сбор метрик
```

Отдельно проверить:

- prompt-to-video;
- image-to-video;
- storyboard-to-video;
- character sheet для стабильного персонажа;
- вертикальный формат для Shorts/Reels;
- ролики для Telegram.

## Что проверить

- Какой именно Google/Veo-сервис использован в источнике.
- Доступен ли он в Германии/ЕС.
- Какие актуальные лимиты бесплатного режима.
- Есть ли watermark.
- Можно ли использовать результат коммерчески.
- Насколько стабильно работает русская озвучка.
- Реально ли storyboard улучшает контроль сцены.
- Можно ли держать одного брендированного персонажа/робота без сильной потери стабильности.
- Какие метрики считать успешным тестом: удержание, CTR, переход, заявка, регистрация, вопрос в личку.

## Как понять, что идея сработала

- Получается сделать 5-10 коротких роликов по разным рекламным гипотезам без ручного тяжелого продакшена.
- Ролики не просто развлекают, а ведут к понятному следующему шагу.
- Метрики показывают, какие углы подачи стоит развивать.
- Один персонаж/робот узнаваемо повторяется в нескольких роликах.
- Появляется шаблон промпта и минимальный production-процесс, который можно перенести в активную wiki.

## Риски или сомнения

- Обход ограничений через смену почты нельзя превращать в рабочее правило.
- Бесплатный доступ, лимиты и условия платформы могут измениться.
- Чужие лица, голоса, бренды, стили и узнаваемые персонажи могут создать правовые риски.
- "Вирусный" формат сам по себе не равен продаже.
- Для финансовых и торговых тем нельзя превращать ролик в гарантию прибыли или отсутствие риска.
- 8 секунд может быть мало для сложного объяснения, поэтому часть роликов должна вести к следующему материалу.
