# Hermes Desktop как GUI-обертка для агентного workflow TiTan Bots

## Оценка ценности

`8/10`

Идея ценная, потому что Hermes Desktop может дать более удобный GUI-слой для агентной работы: запуск из папки проекта, добавление файлов в контекст, терминал, выбор провайдера и несколько агентных инстансов. Оставить в инкубаторе, потому что видео не доказывает, что Hermes лучше текущего Codex-процесса для TiTan Bots, и любые claims автора нужно проверять на реальной задаче.

## Lifecycle

Эта карточка живет в `К разбору/Идеи`.

Это не active/current знание и не ответ для RAG. Идея становится active wiki-карточкой только после проверки, решения владельца и отдельного переноса в нужный слой. Если идея устарела или отклонена, перенести в `Архив/К разбору/Отклонено` или другой закрытый архив входов.

`К разбору/Идеи` и `К разбору/Ошибки` являются отдельными ручными входами после audit-stack. Status/queue/warning/guard не является финальным закрытием без terminal owner или lifecycle destination.

## Идея

Проверить Hermes Desktop как возможный дополнительный интерфейс для AI-agent workflow TiTan Bots, не заменяя текущий Codex-процесс без отдельного решения.

Главный смысл из источника: Hermes Desktop подается как графическая оболочка над агентом, где можно работать из конкретной директории, добавлять файлы в контекст через интерфейс, открывать терминал, переключать провайдеров и создавать отдельные агентные инстансы под разные роли.

## Почему может быть ценно

- может ускорить работу с файлами и контекстом в больших папках;
- может быть удобнее для задач, где нужно вручную добавлять конкретные файлы, изображения или материалы в агентный контекст;
- может дать отдельные агентные инстансы под разные роли без смешивания контекста;
- может помочь сравнить UX Codex app / CLI и Hermes Desktop на одинаковых задачах;
- может быть полезно как fallback-инструмент, если отдельные задачи удобнее делать через GUI.

## К чему относится

- TiTan Bots wiki;
- Codex workflow;
- работа с YouTube/transcript/material imports;
- будущий agent workflow;
- проверка альтернативных интерфейсов для AI-разработки и wiki-maintenance.

## Что проверить

- Можно ли запускать Hermes Desktop из нужной рабочей папки TiTan Bots.
- Удобнее ли добавлять файлы в контекст, чем через текущий Codex workflow.
- Есть ли надежная работа с локальными файлами, терминалом и правами доступа.
- Можно ли безопасно подключить нужные провайдеры без риска утечки ключей.
- Есть ли нормальный контроль diff, проверок и approval-gate.
- Не дублирует ли Hermes Desktop уже закрытые задачи Codex app / CLI / context-mode.
- Подходит ли Hermes для реальных задач:
  - импорт YouTube-идей;
  - обзор большой wiki-карточки;
  - разбор кода;
  - подготовка плана;
  - сравнение нескольких файлов.

## Как понять, что идея сработала

- одна и та же задача выполняется быстрее или чище, чем в текущем Codex-процессе;
- меньше ручного копирования контекста;
- агент не теряет границы проекта и не делает лишних правок;
- сохраняется возможность audit/FAST/verification перед завершением;
- инструмент не добавляет новый хаос в правилах, skills и источниках правды.

## Риски или сомнения

- Заголовок и подача видео рекламные; фразу "Codex больше не нужен" не считать фактом.
- Claims про удобство и качество нужно проверять на TiTan Bots, а не переносить как правило.
- Новый GUI может добавить еще один слой инструментов и путаницу.
- Настройка провайдеров, API-ключей и локальных моделей может создать риск безопасности.
- Если Hermes не дает лучшего diff/approval/verifications, он не должен заменять Codex для критичных правок.
