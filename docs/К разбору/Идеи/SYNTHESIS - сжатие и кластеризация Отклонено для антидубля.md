# SYNTHESIS - сжатие и кластеризация Отклонено для антидубля

## Оценка ценности

6/10.

Идея не срочная, потому что текущий SYNTHESIS уже проверяет реальные файлы `Архив/К разбору/Отклонено`. Но если stop-list вырастет, ручной и машинный антидубль может стать шумным и медленным.

## Lifecycle

Эта карточка живет в `К разбору/Идеи`.

Это не active/current знание и не ответ для RAG. Это future/system-идея для улучшения SYNTHESIS и антидубля. Не создавать отдельный audit-layer без отдельного решения владельца.

## Идея

Позже добавить не ручной индекс, а generated stop-list digest для `Архив/К разбору/Отклонено`.

Слой должен читать реальные карточки `Отклонено`, группировать их по смысловым кластерам и формировать плотный машинный отчет для антидубля SYNTHESIS.

## Почему может быть ценно

- закрытые rejected-идеи не будут повторно предлагаться под новыми названиями;
- SYNTHESIS сможет быстрее видеть крупные кластеры отказов;
- не появится ручной индекс, который нужно поддерживать отдельно;
- владелец сможет видеть, какие типы идей уже отвергались и почему.

## К чему относится

- `Архив/К разбору/Отклонено`;
- `synthesis-v1.ps1`;
- антидубль по title, audit_key и смысловым терминам;
- generated reports в `Система/Индексы`.

## Что проверить

- сколько карточек в `Отклонено` станет порогом для запуска digest;
- нужен ли digest только для SYNTHESIS или также для ручного review;
- какие поля извлекать: title, reason, return condition, duplicate terms, cluster;
- не начнет ли digest подменять реальные rejected-карточки;
- должен ли digest обновляться weekly вместе с audit-stack или только по ручному запуску.

## Как понять, что идея сработала

- SYNTHESIS видит уже отвергнутые идеи даже при переформулировке;
- generated digest не требует ручного редактирования;
- реальные карточки `Отклонено` остаются source of truth;
- повторные предложения rejected-идей уменьшаются.

## Риски или сомнения

- преждевременная автоматизация усложнит систему;
- кластеризация может ошибочно склеить разные идеи;
- ручной индекс `Отклонено` создаст второй источник правды, поэтому предпочтителен generated report/cache.
