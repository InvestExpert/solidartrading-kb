# Реестр тестов трафика

Реестр фактических traffic/CPA/advertising тестов. Не использовать для хранения непроверенных списков источников.

Детальная модель тестов пока не закреплена. Вернуться к структуре после первых реальных цепочек `проект -> оффер -> источник трафика -> метод -> результат`.

## Поля теста

- дата;
- проект;
- источник;
- аудитория;
- входной оффер;
- посадочная точка;
- бюджет или лимит;
- метрики;
- результат;
- решение: продолжить / изменить / остановить / отклонить.

## Шаблон Telegram-посева

Использовать как черновой формат будущего теста, если запускается или готовится конкретное размещение.

```md
## YYYY-MM-DD - Telegram-посев / канал X

- проект:
- источник:
- площадка:
- формат:
- ссылка/метка:
- рекламный текст/креатив:
- расход:
- новые подписчики:
- подписчики 7+ дней:
- цена подписчика:
- цена подписчика 7+ дней:
- retention 7 дней:
- дальнейшие действия:
- результат:
- решение:
```
