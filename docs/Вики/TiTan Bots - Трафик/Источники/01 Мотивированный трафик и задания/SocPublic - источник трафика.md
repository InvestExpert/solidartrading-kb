## Применимо к

GoldenConnect, тесты дешевого мотивированного трафика, проверка прохождения простой воронки через сайт и Telegram.

## Суть

SocPublic рассматривается как букс / площадка микрозаданий. Для GoldenConnect он нужен не как подтвержденный качественный источник аудитории, а как дешевый контролируемый источник действий:

- перейти на страницу захвата;
- открыть Telegram-бота;
- пройти цепочку бота;
- подтвердить спонсора;
- прислать отчет для ручной проверки.

## Лучший текущий формат

Первый приоритет: [стандартные задания](./SocPublic%20-%20%D1%81%D1%82%D0%B0%D0%BD%D0%B4%D0%B0%D1%80%D1%82%D0%BD%D1%8B%D0%B5%20%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D1%8F.md).

Причина: оплата идет за выполненное действие, а не за случайный показ, письмо или клик.

## Форматы внутри SocPublic

| Формат | Приоритет для GoldenConnect | Комментарий |
|---|---|---|
| Стандартные задания | высокий | основной формат первого теста |
| Быстрые задания | позже | проверить после стандартных заданий |
| Реактивные задания | позже | сначала понять механику обычных заданий |
| Посещения | низкий / тест клика | можно проверить загрузку страницы, качество слабее |
| Письма | низкий | слабое внимание, риск мусорного эффекта |
| Контекстная реклама внутри букса | низкий | слабый контроль качества |
| Баннеры | низкий | нужен отдельный креатив, качество неясно |

## Предварительный fit

| Проект | Оценка |
|---|---|
| GoldenConnect | подходит для малого теста бесплатного входа и Telegram-воронки |
| SolidarTrading | высокий риск низкого качества, не стартовый источник |
| Универсальные тесты | полезен как полигон учета, UTM, ручной проверки и антифрода |

## Риски

- исполнитель может делать действие только ради оплаты;
- качество аудитории низкое или смешанное;
- возможны фейковые отчеты и повторяющиеся скриншоты;
- правила площадки могут ограничить сложную цепочку из сайта, бота, канала и чата;
- все условия задания должны быть описаны прямо в SocPublic, а не спрятаны на сайте или в Telegram.

## Правила, важные для запуска

- Не просить логины, пароли, SMS-коды, доступы, документы или банковские данные.
- Не делать задание невыполнимым.
- Не требовать действие до нажатия исполнителем кнопки начала выполнения.
- Отказы и доработки должны иметь конкретный комментарий.
- Если отчет не проверить за 5 суток, он может быть одобрен автоматически.
- Внешний ресурс можно использовать как место выполнения действия, но условия выполнения должны быть в описании задания.
- Если в цепочке есть сайт, бот, канал и чат, формулировать это как одну регистрацию / подтверждение внутри Telegram-бота GoldenConnect, а не как продвижение нескольких объектов.

## Что проверяет первый тест

- сколько людей берут задание;
- сколько отправляют отчет;
- сколько отчетов проходят ручную проверку;
- сколько Telegram-ников подтверждается;
- сколько людей видно в GoldenConnect / Telegram;
- сколько визитов пришло по UTM;
- фактическую цену подтвержденной регистрации.
