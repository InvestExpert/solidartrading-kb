# 17. Мессенджерные и push-базы

## Суть

Групповая карточка первичного реестра источников трафика. Это список для изучения и будущих тестов, а не подтверждение эффективности.

## Макро-группа

[02 Telegram и мессенджеры](./02%20Telegram%20%D0%B8%20%D0%BC%D0%B5%D1%81%D1%81%D0%B5%D0%BD%D0%B4%D0%B6%D0%B5%D1%80%D1%8B%20-%20%D0%98%D0%BD%D0%B4%D0%B5%D0%BA%D1%81.md)

## Первичная оценка

| Поле | Значение |
|---|---|
| Скорость запуска | быстро |
| Примерное качество | зависит от согласия и базы |
| Уровень конкуренции | средняя |
| Риск мусорного трафика | высокий |
| Подходит для теста бесплатной регистрации | да |
| Подходит для платного оффера | осторожно |
| Нужен tracker / redirect / UTM | обязательно |
| Статус | осторожно |

## Источники из первичного списка

- Telegram bot mailings
- VK-рассылки
- email-базы
- browser push базы
- in-page push базы
- Viber рассылки
- WhatsApp Business рассылки — осторожно
- SMS-рассылки — осторожно
- MAX-каналы/рассылки, если актуально
- подписные базы сайтов
- базы расширений браузера

## Дополнено из видео

Источник: [Источники трафика в вертикали финансов 2026](https://www.youtube.com/watch?v=YyVIOvTkNGA), обработано 2026-06-26.

- MAX рассматривать как новый мессенджерный источник с низкой конкуренцией, но незрелой инфраструктурой и неясными правилами. Пока это гипотеза для изучения, а не рабочий канал.
- Посевы в MAX и перелив Telegram -> MAX фиксировать как отдельные тестовые гипотезы, если владелец отдельно подтвердит проект, оффер и legal-check.
- SMS-рассылки учитывать только как opt-in канал с подтвержденным согласием и корректной работой с персональными данными.
- Нелегальные базы, SMS без согласия и обзвон без согласия не относить к рабочим источникам.

Источник: [12 бесплатных источников трафика для бизнеса в 2026](https://www.youtube.com/watch?v=gz1MtJYXcNg), обработано 2026-06-26.

- CRM, email, чат-боты, мобильное приложение и push считать owned audience / retention layer, а не внешним холодным источником.
- Цель owned layer: повторные касания, реактивация базы, поддержание контакта и повторные продажи.
- Проверять только при наличии согласия на коммуникацию и понятной сегментации базы.
- Для traffic-аналитики owned layer связывать с источником первоначального привлечения, иначе нельзя понять, что реально дало качественную аудиторию.

## Форматы трафика

- Форматы в исходном промте не выделены отдельным списком.

## Пометка из исходного промта

разделить легальные opt-in базы и серый спам. Серый спам не заносить в рабочие методы, только в риск/черный список.

## Что проверить

- совпадение аудитории с проектом;
- качество действия после первого перехода;
- риск фрода, накрутки или мотивированного мусора;
- возможность раздельного tracking / UTM;
- применимость к GoldenConnect, SolidarTrading и универсальным тестам отдельно.

## Граница

Не считать источники из этой карточки рабочими до отдельного теста или решения владельца. Не создавать рекламные тексты, офферы и бизнес-обещания на основании этой карточки.
