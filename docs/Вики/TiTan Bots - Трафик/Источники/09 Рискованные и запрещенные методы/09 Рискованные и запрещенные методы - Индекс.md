# 09 Рискованные и запрещенные методы - Индекс

Группа для методов, которые нельзя случайно принять за рабочие источники трафика. Этот раздел нужен как стоп-слой и память о запретах.

## Что входит

- серые базы;
- спам-рассылки;
- инвайтинг без согласия;
- автоспам;
- накрутки;
- мотивированные отзывы;
- клоакинг;
- фейковые аккаунты;
- парсинг контактов для спама;
- навязчивое распространение ссылок через личные контакты, near-field отправку или площадки, где пользователь не давал согласие на рекламный контакт.
- серые аккаунты, платежки, прокси, антидетект и расходники, если они используются для обхода правил площадок.

## Вероятная аудитория

Даже если такие методы дают объем, качество, легальность, репутационный риск и переносимость результата неприемлемы или требуют отдельного запрета.

## Предварительный offer-fit

Не использовать как рабочий offer-fit. Если метод попадает сюда, он не должен становиться рекомендацией для проекта без отдельного пересмотра владельцем.

## Предварительный project-fit

- GoldenConnect: не использовать без отдельного решения владельца.
- SolidarTrading: не использовать без отдельного решения владельца.
- Универсальные тесты: не использовать как обычный тестовый источник.

## Методы внутри группы

Методы фиксируются только для запрета, отклонения или понимания риска. Не превращать описание метода в инструкцию по применению.

## Риски

- нарушение правил площадок или закона;
- репутационный ущерб;
- блокировки аккаунтов и доменов;
- мусорные метрики;
- невозможность использовать результат как нормальное доказательство.
- зависимость результата от обхода правил, а не от нормального product/offer fit.

## Что проверять первым

- почему метод попал в рискованный слой;
- есть ли легальный и этичный аналог;
- нужно ли занести конкретный источник в отклоненные;
- есть ли риск случайного повторения.

## Граница результата

Этот раздел не является списком рабочих источников. Конкретные отклонения фиксируются в реестре отклоненных источников или будущей модели выводов.
