# Схемы из материалов

Реестр схем, извлеченных из видео, конспектов и внешних материалов. Это не реестр наших результатов и не список рабочих связок.

## Правило

Схема из материала сохраняется целиком:

- источник или группа источников;
- аудитория;
- входной оффер или тип оффера;
- посадочная точка;
- промежуточная механика;
- заявленный или ожидаемый результат;
- что проверить;
- риски;
- источник материала.

Если результат указан автором видео, фиксировать его как внешнее утверждение, а не как результат TiTan Bots.

## Схемы

### SCHEME-FIN-001 - Холодный финансовый трафик через витрину

| Поле | Значение |
|---|---|
| Статус | внешняя гипотеза |
| Источник трафика | контекст / VK / Telegram / брокерский или другой финансовый трафик |
| Аудитория | пользователь с финансовым намерением или слабой сегментацией |
| Оффер | набор финансовых офферов, не один прямой оффер |
| Посадочная точка | витрина |
| Механика | источник -> витрина -> фильтрация дублей и неподходящих офферов -> подходящий оффер -> дожим |
| Ожидаемый результат | выше шанс конверсии и меньше показов неподходящих офферов |
| Что проверить | EPC, CR, CPA, жалобы, дубли, downstream-качество |
| Риски | legal/compliance, privacy, переносимость МФО-логики на наши проекты |
| Источник | [Источники трафика в вертикали финансов 2026](https://www.youtube.com/watch?v=YyVIOvTkNGA) |
