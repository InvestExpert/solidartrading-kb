## RAG-контекст

Эта карточка является current owner-card для страниц захвата SolidarTrading как входных точек воронки. Из нее можно извлекать роль страницы, допустимые маршруты, связь с AI-помощником, Telegram, регистрацией, GoldenConnect и концепцией "роботы продают сами себя".

Нельзя извлекать из этой карточки конкретный публичный оффер, гарантии, цифры доходности, финальный CTA, дизайн-стандарт или готовый текст лендинга. Все офферы, цифры, доказательства, обещания и риск-границы должны иметь source_basis в Active Wiki.

## Короткий ответ

Страница захвата - это входная точка для трафика. На нее приходит аудитория, а главная задача страницы - заинтриговать пользователя, захватить внимание и перевести его в следующий шаг проекта.

Страница захвата не обязана полностью объяснять SolidarTrading, быть инструкцией по подключению или закрывать всю продажу на себе. Она создает интерес и направляет человека дальше: к регистрации, AI-помощнику, Telegram, доказательствам, материалам, владельцу / партнеру или будущему GoldenConnect route.

## Роль в системе "роботы продают сами себя"

Рабочая цепочка:

```text
ATLAS создает предмет интереса
-> база фиксирует метод, офферы, риски и доказательства
-> страница захвата принимает трафик и создает интригу
-> AI-помощник объясняет и ведет дальше
-> Telegram / регистрация / owner / партнер / GoldenConnect становятся следующими шагами
-> future traffic agents ведут аудиторию на такие страницы
```

Страница захвата находится между трафиком и объяснением. Она не заменяет базу знаний, AI-помощника, Telegram, владельца, партнера или FAQ.

## Что делает страница захвата

- принимает внешний или внутренний трафик;
- быстро показывает главный интерес;
- удерживает внимание;
- не перегружает человека техническими деталями;
- дает один или несколько понятных следующих шагов;
- направляет человека в правильный маршрут;
- помогает тестировать угол подачи, аудиторию, CTA и первый интерес;
- может быть отдельной точкой для SYNTHESIS-MARKETING-FUNNEL.

## Основные маршруты после страницы

Текущие допустимые маршруты:

- регистрация / стартовый маршрут;
- AI-помощник;
- Telegram-группа / чат;
- проверка доказательств, статистики и дополнительных материалов;
- ручной контакт с владельцем / командой / партнером;
- партнерский маршрут, если пользователь пришел от партнера.

Будущий маршрут:

- GoldenConnect route / выбор формата участия, когда этот маршрут будет готов и публичные формулировки будут утверждены.

## Много страниц, не один шаблон

Страниц захвата может быть много. Они могут отличаться по:

- стилю;
- длине;
- аудитории;
- офферу;
- визуалу;
- CTA;
- уровню объяснения;
- входному каналу;
- следующему шагу.

Одна страница не является обязательным стандартом для всех будущих страниц. Текущая сказочная версия - один из вариантов подачи, а не универсальный шаблон.

## Текущая сказочная версия

Текущая сказочная версия фиксируется как один из возможных углов:

- простая, мемная, сказочная подача;
- без технических деталей;
- без Forex-терминов;
- на широкую аудиторию среднего и ниже уровня входа;
- главный смысл: большие деньги были рядом, но человек проходил мимо; есть метод, к которому можно присоединиться;
- цель - заинтриговать и отправить дальше в проект.

Эта версия не должна становиться единственным каноном для всех будущих страниц.

## Source basis rule

Страница захвата использует только те офферы, цифры, доказательства, обещания и риск-границы, которые имеют source_basis в Active Wiki.

Основные владельцы смысла:

- продукт / текущий фокус - карточки по ATLAS и SolidarTrading method;
- доказательства - proof-карты ATLAS;
- риски - risk-boundary карточки;
- формулировки - банк сильных формулировок;
- следующий шаг - FAQ, AI-помощник, регистрация, Telegram, партнерка, GoldenConnect;
- аудитория - карточка целевой аудитории и форматов участия.

Если для конкретного оффера, цифры, CTA или обещания нет source_basis, страница не должна использовать это как факт.

## Чего не делать

- не превращать страницу захвата в длинную техническую инструкцию;
- не перегружать деталями подключения;
- не смешивать все маршруты в одну кашу;
- не использовать офферы без source_basis;
- не обещать то, что запрещено risk-карточками;
- не фиксировать один дизайн как обязательный шаблон;
- не делать вид, что страница сама полностью продает проект без AI, Telegram, владельца, партнера или следующего шага;
- не обещать гарантированную конверсию;
- не обещать гарантированный доход;
- не использовать будущие боты как current marketing direction;
- не утверждать, что GoldenConnect route уже полностью готов, если финальные условия еще не утверждены.
