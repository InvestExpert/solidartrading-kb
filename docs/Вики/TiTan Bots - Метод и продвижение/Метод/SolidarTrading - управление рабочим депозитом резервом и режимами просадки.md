## RAG-контекст

Эта карточка - публичный канон для участника по рабочему депозиту, резерву и режимам состояния счета.

Не использовать как персональную финансовую рекомендацию. Карточка объясняет логику системы и общий ориентир. Решение о пополнении, выводе, размере резерва и личном риске принимает сам участник.

Внутренние правила команды по master-счету, точные действия с резервом и операционные режимы не являются командой участнику. В публичном ответе отделять "что делает команда" от "на что может ориентироваться участник".

## Короткий ответ

Рабочий депозит в SolidarTrading нельзя оценивать только по проценту на мониторинге. Для понимания состояния счета важны фактические данные брокерского счета: баланс, средства, открытый минус, открытые позиции и свободная маржа.

Публичная формулировка:

```text
Мы смотрим не только на процент просадки, а на фактическое состояние счета: баланс, средства, открытый минус, позиции и свободную маржу. По этим данным определяется текущий режим счета. Участник может ориентироваться на этот режим, но решение о своих деньгах принимает сам.
```

## Что такое рабочий депозит

Рабочий депозит - это сумма, которая участвует в работе стратегии. Ее нельзя воспринимать как полностью свободный остаток, особенно если на счете есть открытые позиции или текущая просадка.

Для публичного объяснения базовая рабочая единица SolidarTrading:

```text
250 USD
```

Эта сумма используется как простой ориентир для понимания масштаба счета, резерва и риска. Она не является персональной рекомендацией держать на счете ровно такую сумму.

## Старт для Profit и Safe

Рекомендованный старт для публичных стратегий ATLAS:

- `TiTan ATLAS Profit` - ориентироваться на старт `от 250 USD`.
- `TiTan ATLAS Safe` - основной рекомендованный старт также `от 250 USD`.

Для `Safe` около `100 USD` теоретически возможно, потому что конфигурация рассчитана на более осторожный старт. Но это не основной рекомендованный вариант.

Короткая формулировка:

```text
Safe можно рассматривать для более осторожного старта, но резерв все равно важен. Рекомендованный старт - от 250$. Около 100$ теоретически возможно, но чем меньше сумма, тем важнее иметь резерв.
```

Не использовать маленький депозит как обещание безопасности. Малый счет может легче почувствовать комиссию, спред, просадку, момент подключения и любую нестандартную рыночную ситуацию.

## Что такое резерв

Резерв - это деньги вне текущей рабочей нагрузки счета. Он нужен не для обещания полной безопасности, а как запас гибкости.

Резерв может быть полезен, когда:

- счет входит в зону повышенной просадки;
- участнику нужно не принимать решения на эмоциях;
- команда публикует статус счета и объясняет текущий режим;
- участник хочет заранее понимать, какие действия возможны при ухудшении или стабилизации ситуации.

Резерв не означает, что любая просадка будет безопасно пережита.

## Как понимать просадку

Для публичного объяснения важно разделять:

- мониторинг доходности и просадки;
- фактическое состояние брокерского счета;
- открытый минус по текущим позициям;
- рабочий режим счета.

Myfxbook или другой публичный мониторинг может показывать динамику, но не должен быть единственным основанием для решений. Состояние счета нужно понимать по фактическим данным брокера.

Простая формула для объяснения открытого минуса:

```text
Открытый минус = Balance - Equity
```

Если `Equity` выше `Balance`, по этой формуле открытого минуса нет; состояние все равно нужно смотреть по открытым позициям и текущему режиму счета.

## Режимы счета простым языком

Публично режимы счета можно объяснять цветами:

| Цвет | Смысл для участника |
|---|---|
| Зеленый | счет в штатной зоне, специальных действий не требуется |
| Желтый | зона повышенного внимания, лучше не делать хаотичных действий |
| Оранжевый | команда может принимать защитные решения по master-счету |
| Красный | высокая зона риска, нужен частый контроль со стороны команды |
| Ручной | механические обещания недопустимы, решения принимает команда по фактической ситуации |

Для участника это ориентир, а не команда пополнять или выводить деньги.

## Долив и вывод

Публично можно говорить:

- долив может быть защитным инструментом, но не гарантирует спасение счета;
- вывод во время просадки может ослабить счет;
- решение о своих деньгах принимает сам участник;
- нестандартные действия лучше обсуждать в общем чате;
- команда может сообщать, что делает по master-счету, но это не является персональной командой участникам.

Нельзя говорить:

- `всем срочно долить`;
- `всем вывести резерв`;
- `долив точно решит проблему`;
- `так безопасно для каждого`;
- `просадка гарантированно восстановится`.

## Пульс счета для Telegram

Для Telegram-чата нужен понятный публичный статус счета.

Рабочие названия формата:

- `пульс счета`;
- `здоровье счета`;
- `температура счета`.

Статус должен включать:

- текущий цвет режима;
- текущую рабочую просадку по фактическим данным;
- короткое объяснение состояния;
- что делает команда;
- что участник может использовать только как ориентир;
- ссылку на эту инструкцию.

## Что говорить участнику

Можно:

- объяснять режим счета;
- показывать общую логику рабочей суммы и резерва;
- говорить, что резерв нужен как запас гибкости;
- объяснять разницу между рекомендуемым стартом `250 USD` и теоретическим малым стартом для `Safe`;
- говорить, что участник может ориентироваться на публичный статус;
- напоминать, что решение о личных деньгах принимает сам участник.

Нельзя:

- говорить конкретному человеку, сколько именно ему долить или вывести;
- превращать действия команды по master-счету в обязательную команду для всех;
- обещать, что долив спасет счет;
- подавать старт около `100 USD` в `Safe` как основной рекомендуемый вариант;
- считать Myfxbook-проценты достаточной основой для решения;
- публиковать статус как гарантию безопасности.
