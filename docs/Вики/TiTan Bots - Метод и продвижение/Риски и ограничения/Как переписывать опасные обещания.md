# Как переписывать опасные обещания

## RAG-контекст

Эта карточка - внутреннее правило редакторской переработки опасных обещаний, а не публичный ответ пользователю.

RAG может использовать ее для безопасной переформулировки, но не должен выдавать сам процесс импорта как клиентский текст.

## Суть

Опасную рекламную фразу не надо сразу удалять. В ней может быть хороший крючок.

Правильный процесс:

1. Сохранить оригинал.
2. Отметить риск.
3. Понять, какую выгоду хотели сказать.
4. Дать 3-5 рабочих замен.
5. Не превращать замену в скучный дисклеймер.

## Типовые замены

### Жесткий запрет

Не использовать как публичное обещание:

- `гарантированно`;
- `без риска`;
- `100% безопасно`;
- `точно заработаете`;
- `доход без просадки`;
- `пассивный доход без риска`.

Сильную фразу не нужно убивать, но ее нужно переписать так, чтобы рядом были источник, период, риск и граница обещаний.

### "Без риска"

Не использовать как прямое обещание.

Варианты:

- с контролем риска;
- с умеренным риском;
- с управляемым риском;
- риск остается, но система берет его под контроль;
- меньше ручной торговли, больше автоматизации;
- без передачи денег в чужие руки.

### "Гарантированный доход"

Варианты:

- цель системы - регулярная автоматизированная торговля;
- метод рассчитан на долгосрочную работу;
- результат зависит от рынка, настроек и размера счета;
- фокус на системе, а не на угадывании рынка.

### "ИИ взломал код прибыли"

Варианты:

- ИИ помогает превратить торговлю из хаоса в систему;
- технологии берут рутину рынка на себя;
- автоматизация помогает действовать по правилам, а не по эмоциям;
- торговый робот работает там, где человек устает.

### "Ежеминутное начисление прибыли"

Варианты:

- результат счета меняется вместе с работой рынка;
- баланс и сделки можно отслеживать онлайн;
- движение счета видно в реальном времени;
- прибыль не начисляется по расписанию, она появляется только как результат сделок.

### "Стабильный доход"

Варианты:

- долгосрочный подход;
- системная работа;
- регулярная торговая активность;
- цель - сгладить хаос ручной торговли;
- стабильность оценивается по серии периодов, а не по одному дню.

### "Пассивный доход"

Можно использовать как описание формата участия: пользователь не торгует вручную, а подключается к автоматизации или копированию сделок.

Нельзя использовать как обещание дохода без риска, контроля и ответственности.

Варианты:

- участие без ручной торговли;
- автоматизированная торговля на счете пользователя;
- пользователь не торгует вручную, но рыночный риск остается;
- результат зависит от рынка, настроек, счета и периода.

### "Безопасно" и "деньги в безопасности"

Можно использовать только в точном смысле:

- деньги не передаются проекту;
- деньги остаются на счете или кошельке пользователя;
- есть настройки риска, мониторинг и контроль;
- это безопаснее, чем отдавать деньги внешнему управляющему без контроля.

Нельзя использовать как обещание отсутствия рыночного риска или невозможности потерять депозит.

Рабочая граница риска:

```text
Существует риск потери капитала, как и на любом финансовом рынке.
```

## Правило для импорта

При импорте маркетингового материала опасные фразы сохранять в банк формулировок или страницу рисков.

Для каждой опасной фразы сразу писать безопасные варианты, чтобы потом не начинать с нуля.
