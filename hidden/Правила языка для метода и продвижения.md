# Правила языка для метода и продвижения

## Главный принцип

Материалы слоя [TiTan Bots - Метод и продвижение](undefinedhidden/TiTan%20Bots%20-%20%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%20%D0%B8%20%D0%BF%D1%80%D0%BE%D0%B4%D0%B2%D0%B8%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5%20-%20%D0%98%D0%BD%D0%B4%D0%B5%D0%BA%D1%81.md) пишутся для обычного человека, а не для разработчика.

Сначала выгода. Потом объяснение. Потом факты, доказательства и ограничения.

## Запрещено

- писать сухим техническим языком;
- начинать с терминов, классификаций и архитектуры;
- использовать английские слова без крайней необходимости;
- писать для "своих", если текст должен продавать или объяснять;
- прятать выгоду в конце страницы.

## Если термин нужен

Если термин нельзя убрать, дать простой перевод или аналог в скобках.

Плохо:

```text
copy trading model
```

Хорошо:

```text
копитрейдинг (автоматическое повторение сделок на счете клиента)
```

Плохо:

```text
market monetization engine
```

Хорошо:

```text
бот, который старается зарабатывать на самом движении цены, а не на угадывании направления
```

## Порядок продажной страницы

1. Боль, желание или интрига.
2. Главная выгода для человека.
3. Простое объяснение идеи.
4. Почему это может работать.
5. Доказательства или что нужно доказать.
6. Ограничения, риски и аккуратные формулировки.

## Стиль

Можно использовать:

- интригу;
- сильные заголовки;
- контраст "как обычно" против "как у нас";
- короткие фразы;
- эпатаж, если он не превращается в ложное обещание;
- понятные образы вместо терминов.

Нельзя превращать эпатаж в гарантию дохода.

## Пример

Плохо:

```text
FLAT позиционируется как hybrid grid + hedge / fork logic.
```

Хорошо:

```text
FLAT - бот для тех, кто не хочет угадывать рынок вручную.

Он не спрашивает: "куда пойдет цена?"
Он работает иначе: использует движение рынка по заранее заданным правилам.

Для клиента смысл простой: меньше ручной торговли, больше автоматизации.
```
